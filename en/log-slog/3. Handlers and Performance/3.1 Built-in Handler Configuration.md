# Built-in Handler Configuration

Go's built-in [`slog.TextHandler`](https://pkg.go.dev/log/slog#TextHandler) and [`slog.JSONHandler`](https://pkg.go.dev/log/slog#JSONHandler) handle more than just rendering log entries as text or JSON. Via [`slog.HandlerOptions`](https://pkg.go.dev/log/slog#HandlerOptions), they can capture source call sites, filter log levels, and transform or sanitize attributes before entries are output.

These settings are configured once when instantiating the handler and apply globally to all events routed through it. Application code continues using standard `*slog.Logger` calls without coupling itself to underlying field names or output formats.

## Capabilities of HandlerOptions

The `slog.HandlerOptions` struct contains three fields:

| Field | Purpose |
| :--- | :--- |
| `AddSource` | Appends source code location (function name, file path, and line number) |
| `Level` | Sets the minimum enabled severity level |
| `ReplaceAttr` | Transforms, renames, or drops attributes prior to formatting |

A zero-value `HandlerOptions` struct yields standard default behavior: `AddSource` is false, minimum level defaults to `Info`, and attributes are written without transformation. Passing `nil` as options to `NewJSONHandler` or `NewTextHandler` uses this same default configuration.

Level thresholds and dynamic adjustments via `slog.LevelVar` were covered in [“Levels and Filtering”](/en/log-slog/intro/levels-and-filtering). This article focuses on `AddSource` and `ReplaceAttr`.

## Capturing Source Code Location

Setting `AddSource: true` instructs the handler to attach a `source` attribute to every log record:

```go
handler := slog.NewJSONHandler(
    os.Stdout,
    &slog.HandlerOptions{
        AddSource: true,
    },
)
logger := slog.New(handler)

logger.Error("load configuration failed")
```

The resulting JSON output includes source location metadata:

```json
{"time":"2026-07-22T12:00:00Z","level":"ERROR","source":{"function":"main.main","file":"/app/cmd/service/main.go","line":24},"msg":"load configuration failed"}
```

The [`slog.Source`](https://pkg.go.dev/log/slog#Source) struct contains the function name, absolute file path, and line number. While helpful for tracking down origin lines in code, file paths increase payload size and depend on local build paths, making them unideal as high-cardinality aggregation keys.

Note that `AddSource` records the frame where `logger.Error` or `logger.Info` was called. If you wrap logger calls inside a helper function, `source` will point to the location inside your helper rather than the outer caller site. Capturing true caller sites across custom helper layers requires working directly with `slog.Record` and falls outside standard handler configuration.

`AddSource` applies globally to all enabled log levels. If you need source call sites attached selectively (such as only for `Error` events), you must use a custom handler or attribute transformer.

## Standard System Attributes

Built-in handlers inject default system attributes into every record regardless of application fields:

| Constant | Default Key | Content |
| :--- | :--- | :--- |
| [`slog.TimeKey`](https://pkg.go.dev/log/slog#TimeKey) | `time` | Event creation timestamp |
| [`slog.LevelKey`](https://pkg.go.dev/log/slog#LevelKey) | `level` | Severity level |
| [`slog.MessageKey`](https://pkg.go.dev/log/slog#MessageKey) | `msg` | Log event message string |
| [`slog.SourceKey`](https://pkg.go.dev/log/slog#SourceKey) | `source` | Source call site (when `AddSource` is true) |

Use these package constants rather than string literals when matching system keys in handler callbacks. For example, centralized log platforms often require `@timestamp`, `severity`, and `message` instead of Go's default `slog` key names.

Avoid reusing these system key names at the top level of application attribute calls. `JSONHandler` does not deduplicate keys, and external log parsers handle duplicate JSON keys inconsistently.

## Attribute Transformation via ReplaceAttr

The `ReplaceAttr` field accepts a callback function invoked before rendering each non-group attribute:

```go
func replaceAttr(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    return attr
}
```

The `attr` argument arrives pre-resolved: if an attribute value implements `slog.LogValuer`, the handler calls `LogValue()` before passing the attribute to `ReplaceAttr`. The function can return the attribute unmodified, return a modified `slog.Attr`, or return a zero-value `slog.Attr{}` to drop the attribute entirely.

The `groups` parameter contains a slice of open group names enclosing the attribute. For top-level attributes, `groups` is empty. The `groups` slice is owned by the handler and must not be mutated or retained beyond the callback invocation.

### Renaming Standard System Keys

Checking `len(groups) == 0` ensures you modify top-level system attributes without accidentally renaming matching fields inside nested application groups:

```go
func renameBuiltinAttrs(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) != 0 {
        return attr
    }

    switch attr.Key {
    case slog.TimeKey:
        attr.Key = "@timestamp"
    case slog.LevelKey:
        attr.Key = "severity"
    case slog.MessageKey:
        attr.Key = "message"
    case slog.SourceKey:
        attr.Key = "caller"
    }

    return attr
}
```

Assign this callback to `HandlerOptions` when instantiating the handler:

```go
options := &slog.HandlerOptions{
    AddSource:   true,
    ReplaceAttr: renameBuiltinAttrs,
}

handler := slog.NewJSONHandler(os.Stdout, options)
logger := slog.New(handler)
```

Application log calls remain unchanged, while emitted JSON records adhere to downstream schema requirements:

```json
{"@timestamp":"2026-07-22T12:00:00Z","severity":"INFO","caller":{"function":"main.main","file":"/app/cmd/service/main.go","line":31},"message":"service started"}
```

### Modifying Attribute Values

`ReplaceAttr` can modify an attribute's value type as well as its key name. The following transformer converts the system timestamp to Unix epoch milliseconds:

```go
func replaceTime(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 0 && attr.Key == slog.TimeKey {
        attr.Value = slog.Int64Value(
            attr.Value.Time().UnixMilli(),
        )
    }

    return attr
}
```

This transformation is useful when integrating with log systems that require numeric timestamps. For human readability, standard ISO-8601/RFC-3339 strings remain preferable as they preserve explicit time zone offsets.

For `slog.TimeKey`, the value is guaranteed to be `time.Time`. When inspecting general application attributes, check [`slog.Value.Kind`](https://pkg.go.dev/log/slog#Value.Kind) before calling type-specific getters like `Value.Time()` to avoid panics.

### Group-Aware Replacements

`ReplaceAttr` is not called for group attributes themselves, but it is called for every leaf attribute enclosed within a group. Given this call:

```go
logger.Info("upstream request completed",
    slog.Group("http",
        slog.Int("status", 200),
        slog.Group("upstream",
            slog.String("host", "api.example.com"),
        ),
    ),
)
```

The handler passes the following `groups` paths to `ReplaceAttr`:

| Attribute Key | `groups` Slice |
| :--- | :--- |
| System `time`, `level`, `msg` | `[]string{}` (empty) |
| `status` | `[]string{"http"}` |
| `host` | `[]string{"http", "upstream"}` |

Inspecting the `groups` slice allows scoping attribute transformations to specific namespaces:

```go
func renameHTTPStatus(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 1 &&
        groups[0] == "http" &&
        attr.Key == "status" {
        attr.Key = "status_code"
    }

    return attr
}
```

Checking only `attr.Key` without validating `groups` risks modifying identical key names that belong to different groups (e.g., `http` vs `database`).

### Omitting Attributes

Returning a zero-value `slog.Attr{}` strips an attribute from the output record entirely. For instance, to remove the system timestamp:

```go
func removeTime(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 0 && attr.Key == slog.TimeKey {
        return slog.Attr{}
    }

    return attr
}
```

Omit timestamps only if an external log collector automatically attaches ingestion timestamps. In unit tests, un-marshal JSON output into a test struct and omit the `time` field from assertions rather than stripping timestamps at the handler level.

## Data Redaction and Field Sanitization

`ReplaceAttr` can act as a fallback safety layer to sanitize sensitive keys across log calls:

```go
func redactSensitiveAttr(
    _ []string,
    attr slog.Attr,
) slog.Attr {
    switch strings.ToLower(attr.Key) {
    case "authorization", "password", "token":
        return slog.String(attr.Key, "[REDACTED]")
    default:
        return attr
    }
}
```

Because `ReplaceAttr` receives attributes individually, it redacts matching keys inside `slog.Group` blocks as well. However, passing an entire struct via `slog.Any("request", req)` yields a single attribute whose value is a struct; `ReplaceAttr` does not traverse internal struct fields reflectively and cannot redact nested fields like `req.Password`. Nor will it sanitize sensitive data embedded directly inside message strings or error messages.

Use `ReplaceAttr` sanitization as a defense-in-depth safety net rather than a primary redaction mechanism. Application code should construct explicit safe attribute lists, and domain types should implement `LogValuer` as detailed in [“Custom Values”](/en/log-slog/events/custom-values).

## Comprehensive Handler Setup

Multiple transformation rules can be combined into a single, clean handler constructor:

```go
func replaceLogAttr(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 0 {
        switch attr.Key {
        case slog.TimeKey:
            attr.Key = "@timestamp"
        case slog.LevelKey:
            attr.Key = "severity"
        case slog.MessageKey:
            attr.Key = "message"
        case slog.SourceKey:
            attr.Key = "caller"

            if source, ok := attr.Value.Any().(*slog.Source); ok {
                source.File = filepath.Base(source.File)
            }
        }
    }

    switch strings.ToLower(attr.Key) {
    case "authorization", "password", "token":
        return slog.String(attr.Key, "[REDACTED]")
    }

    return attr
}

func newLogger(
    out io.Writer,
    level slog.Leveler,
) *slog.Logger {
    handler := slog.NewJSONHandler(
        out,
        &slog.HandlerOptions{
            AddSource:   true,
            Level:       level,
            ReplaceAttr: replaceLogAttr,
        },
    )

    return slog.New(handler)
}
```

Here, source file paths are trimmed to base file names using [`filepath.Base`](https://pkg.go.dev/path/filepath#Base) while retaining function names and line numbers. The `level` parameter accepts either `slog.Level` or `*slog.LevelVar`, keeping level filtering flexible without leaking into domain code.

Because `ReplaceAttr` runs on every emitted record, it must be fast, lock-free, and thread-safe. Avoid network I/O, heavy allocations, or shared state mutations inside `ReplaceAttr`.

Built-in handler options suffice when transformations depend strictly on individual attributes and group paths. For advanced requirements—such as inspecting entire records, extracting values from `context.Context`, fanning out logs to multiple outputs, or conditional multi-field logic—use a handler wrapper or implement a custom `slog.Handler`.
