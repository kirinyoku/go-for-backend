# Client Initialization

The [`net/http`](https://pkg.go.dev/net/http) package provides two levels for making outgoing HTTP requests: package-level helpers such as [`http.Get`](https://pkg.go.dev/net/http#Get) for quick starts, and the [`http.Client`](https://pkg.go.dev/net/http#Client) type for explicit control over client behavior.

`http.Client` does not open connections directly. It has a [`Transport`](https://pkg.go.dev/net/http#Transport) field, which delegates request execution to the network layer: opening connections, TLS, proxies and TCP connection reuse.

## Package-level Functions

For short examples, scripts and manual checks, package-level functions such as [`http.Get`](https://pkg.go.dev/net/http#Get), [`http.Post`](https://pkg.go.dev/net/http#Post) and [`http.Head`](https://pkg.go.dev/net/http#Head) are convenient. They let you make a request without creating a client explicitly.

```go
package main

import "net/http"

func fetchStatus(url string) (int, error) {
    resp, err := http.Get(url)
    if err != nil {
        return 0, err
    }
    defer resp.Body.Close()

    return resp.StatusCode, nil
}
```

## Limits of http.DefaultClient

The `http.Get`, `http.Post` and `http.Head` functions use the global [`http.DefaultClient`](https://pkg.go.dev/net/http#DefaultClient).

```go
var DefaultClient = &Client{}
```

`http.DefaultClient` is a client with default settings. If its `Transport` field is `nil`, the network layer uses the global [`http.DefaultTransport`](https://pkg.go.dev/net/http#DefaultTransport).

This is convenient for small scripts, but applications usually use an explicitly configured client. The main reason is control over behavior. For example, `http.DefaultClient` has a zero `Timeout`, so there is no overall deadline for the request.

## Configuring http.Client

The `http.Client` struct lets you define settings once and then make requests through that client's methods.

```go
package main

import (
    "net/http"
    "time"
)

var client = &http.Client{
    Timeout: 10 * time.Second,
}
```

After initialization, a simple `GET` request can be made with [`client.Get`](https://pkg.go.dev/net/http#Client.Get):

```go
func fetchStatus(client *http.Client, url string) (int, error) {
    resp, err := client.Get(url)
    if err != nil {
        return 0, err
    }
    defer resp.Body.Close()

    return resp.StatusCode, nil
}
```

This code keeps the same convenient flow as `http.Get`, but the client's behavior is now explicit. The differences between [`client.Timeout`](https://pkg.go.dev/net/http#Client.Timeout), context deadlines and `Transport` timeouts are covered in [Timeouts and Context](/en/net-http/client/timeouts-and-context).

## Client Lifecycle

`http.Client` is designed for reuse and is safe for concurrent use by multiple goroutines. In most applications, create it once and pass it to the code that performs outgoing requests.

```go
type APIClient struct {
    httpClient *http.Client
}

func NewAPIClient() *APIClient {
    return &APIClient{
        httpClient: &http.Client{
            Timeout: 10 * time.Second,
        },
    }
}
```

::: warning
Creating a new `http.Client` with a new `http.Transport` for every request prevents connection reuse. This is covered in more detail in `Connection Pooling`.
:::
