# Transport Layer Configuration

While [`http.Client`](https://pkg.go.dev/net/http#Client) controls high-level request behavior, the transport layer handles the network work. The [`http.RoundTripper`](https://pkg.go.dev/net/http#RoundTripper) interface is the foundation of this system: it is where a client configures TCP connections, TLS, proxies and additional behavior around sending a request.

## The RoundTripper Interface

`RoundTripper` is a single-method interface: [`RoundTrip`](https://pkg.go.dev/net/http#RoundTripper.RoundTrip). It accepts a request and returns a response. The default implementation is [`*http.Transport`](https://pkg.go.dev/net/http#Transport).

```go
type RoundTripper interface {
    RoundTrip(*Request) (*Response, error)
}
```

::: info
`RoundTrip` should not interpret the response, such as checking the status code. Its job is only to deliver the request and return either a response or a network-level error.
:::

## Configuring http.Transport

`http.Transport` is a complex type that manages the connection pool and keep-alive behavior. When changing its settings, it is usually best to **clone** [`http.DefaultTransport`](https://pkg.go.dev/net/http#DefaultTransport), so you keep the standard library's safe defaults, including connection and TLS timeouts.

```go
transport := http.DefaultTransport.(*http.Transport).Clone()
transport.DisableCompression = true // Disable automatic gzip decompression.

client := &http.Client{
    Transport: transport,
}
```

### Useful Settings

| Field | Purpose |
| :---- | :------ |
| [`ResponseHeaderTimeout`](https://pkg.go.dev/net/http#Transport.ResponseHeaderTimeout) | Limits how long the client waits for response headers from the server. |
| [`DisableCompression`](https://pkg.go.dev/net/http#Transport.DisableCompression) | Disables automatic decompression of compressed responses (`gzip`). |
| [`Proxy`](https://pkg.go.dev/net/http#Transport.Proxy) | Routes requests through a proxy server. |
| [`TLSClientConfig`](https://pkg.go.dev/net/http#Transport.TLSClientConfig) | Configures TLS settings and certificate verification. |

::: tip
Connection pool settings such as `MaxIdleConns`, `MaxIdleConnsPerHost` and related fields are covered in `Connection Pooling`.
:::

## RoundTripper Wrappers

Because `http.Client` sends requests through its `Transport` field, you can plug in your own `RoundTripper` implementation. A common pattern is to wrap one transport with another: the outer layer changes or logs the request, then passes it to the real network transport.

This is a convenient way to add cross-cutting behavior for every request made by a specific client: authorization, logging, metrics or retries. The actual network I/O still belongs to the lower transport, usually `http.DefaultTransport` or a configured `*http.Transport`.

### Example: Adding an API Key

```go
type AuthTransport struct {
    APIKey string
    Next   http.RoundTripper
}

func (t *AuthTransport) RoundTrip(req *http.Request) (*http.Response, error) {
    // Clone the request before modifying headers.
    req = req.Clone(req.Context())
    req.Header.Set("X-API-Key", t.APIKey)

    next := t.Next
    if next == nil {
        next = http.DefaultTransport
    }

    return next.RoundTrip(req)
}

func main() {
    client := &http.Client{
        Transport: &AuthTransport{
            APIKey: "secret",
            Next:   http.DefaultTransport,
        },
    }
    // Every request made by this client now contains the API key.
}
```
