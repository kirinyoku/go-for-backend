# HTTPS

The `net/http` package can run an HTTPS server without an external web server. It uses the same [`http.Server`](https://pkg.go.dev/net/http#Server), but calls [`ListenAndServeTLS`](https://pkg.go.dev/net/http#Server.ListenAndServeTLS) instead of [`ListenAndServe`](https://pkg.go.dev/net/http#Server.ListenAndServe).

Here, HTTPS means that HTTP runs over TLS. TLS settings are handled by the [`crypto/tls`](https://pkg.go.dev/crypto/tls) package, and `net/http` uses that configuration when accepting connections.

## Minimal HTTPS Server

The `ListenAndServeTLS` method accepts two paths:

- `certFile`: the server certificate file;
- `keyFile`: the private key file.

```go
package main

import (
    "errors"
    "log"
    "net/http"
    "time"
)

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("GET /", func(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("hello over https\n"))
    })

    server := &http.Server{
        Addr:              ":443",
        Handler:           mux,
        ReadHeaderTimeout: 2 * time.Second,
        ReadTimeout:       5 * time.Second,
        WriteTimeout:      10 * time.Second,
        IdleTimeout:       120 * time.Second,
    }

    err := server.ListenAndServeTLS("cert.pem", "key.pem")
    if err != nil && !errors.Is(err, http.ErrServerClosed) {
        log.Fatalf("https server failed: %s", err)
    }
}
```

`ListenAndServeTLS` loads the certificate and key from files, creates a TLS listener and starts serving HTTPS requests. Like `ListenAndServe`, this method always returns a non-nil error. During a normal shutdown through `Shutdown`, it returns [`http.ErrServerClosed`](https://pkg.go.dev/net/http#ErrServerClosed).

::: info
On Unix-like systems, port `:443` usually requires elevated privileges. Local examples often use a non-public port such as `:8443`.
:::

## Certificate and Private Key

The certificate proves the server's name, and the private key is used during the TLS handshake. The files have to match: the certificate must have been issued for the corresponding private key.

```text
cert.pem  # server certificate, usually PEM-encoded
key.pem   # private key, usually PEM-encoded
```

For a public domain, the certificate is usually issued by a trusted certificate authority. For local experiments, you can use a self-signed certificate, but regular browsers and HTTP clients will not trust it without additional configuration.

`ListenAndServeTLS` accepts file paths specifically. If certificates need to be loaded another way, for example from a secret store or rotated without restarting the process, the usual approach is to configure [`http.Server.TLSConfig`](https://pkg.go.dev/net/http#Server.TLSConfig) and run the server through a TLS listener.

## Configuring TLSConfig

The `TLSConfig` field lets you pass an explicit TLS configuration to the server. For a basic HTTPS server, the defaults are often enough, but the minimum protocol version is sometimes pinned explicitly.

```go
package main

import (
    "crypto/tls"
    "errors"
    "log"
    "net/http"
    "time"
)

func main() {
    mux := http.NewServeMux()

    server := &http.Server{
        Addr:    ":443",
        Handler: mux,
        ReadHeaderTimeout: 2 * time.Second,
        TLSConfig: &tls.Config{
            MinVersion: tls.VersionTLS12,
        },
    }

    err := server.ListenAndServeTLS("cert.pem", "key.pem")
    if err != nil && !errors.Is(err, http.ErrServerClosed) {
        log.Fatalf("https server failed: %s", err)
    }
}
```

In this example, the server accepts TLS 1.2 and newer. TLS 1.3 will be used automatically if the client supports it.

::: warning
Do not copy random TLS settings from old examples. Go's defaults already reflect modern secure settings. Manual configuration of `CipherSuites`, `CurvePreferences` and other low-level fields is only needed when you have a clear compatibility policy.
:::

## Certificates in TLSConfig

If you do not want to pass file paths to `ListenAndServeTLS`, you can load the certificate yourself and pass it through `TLSConfig.Certificates`.

```go
cert, err := tls.LoadX509KeyPair("cert.pem", "key.pem")
if err != nil {
    log.Fatalf("load certificate: %s", err)
}

server := &http.Server{
    Addr:    ":443",
    Handler: mux,
    TLSConfig: &tls.Config{
        MinVersion:   tls.VersionTLS12,
        Certificates: []tls.Certificate{cert},
    },
}

err = server.ListenAndServeTLS("", "")
if err != nil && !errors.Is(err, http.ErrServerClosed) {
    log.Fatalf("https server failed: %s", err)
}
```

Empty strings in `ListenAndServeTLS("", "")` are valid if certificates are already set in `TLSConfig`. In this version, `net/http` does not load certificate files itself; it uses the prepared TLS configuration.

## HTTP/2

When the server runs with TLS, `net/http` automatically enables HTTP/2 support for clients that negotiate that protocol through ALPN. In most cases, this does not require any extra code.

If you need full control over the protocol set, configure it through the server's TLS settings. For a regular HTTPS server, it is better to keep the default behavior.
