# Reading and Writing Data

A web server constantly reads data from requests and writes data to responses: it accepts query parameters, decodes JSON, processes HTML forms, receives files and returns results to clients. In Go, this work is built around the [`io.Reader`](https://pkg.go.dev/io#Reader) and [`io.Writer`](https://pkg.go.dev/io#Writer) interfaces. The streaming model makes it possible to process request bodies without loading them fully into memory. Choosing the right mechanism for each case affects both performance and server security.

## URL Query Parameters

Query string parameters are available through [`r.URL.Query()`](https://pkg.go.dev/net/url#URL.Query), which returns [`url.Values`](https://pkg.go.dev/net/url#Values), an alias for `map[string][]string`.

| Method                                                      | Behavior |
| :---------------------------------------------------------- | :------- |
| [`values.Get("key")`](https://pkg.go.dev/net/url#URL.Query) | Returns the first value for the key, or an empty string. |
| `values["key"]`                                             | Returns all values for the key as a slice. |

```go
func searchHandler(w http.ResponseWriter, r *http.Request) {
    query := r.URL.Query()

    // Get returns the first value for the key, or an empty string.
    q := query.Get("q")

    // All values for one key: ?tag=go&tag=web -> ["go", "web"].
    tags := query["tag"]

    // Query values are strings.
    // Numeric parameters require explicit conversion.
    page, err := strconv.Atoi(query.Get("page"))
    if err != nil || page < 1 {
        // Default value.
        page = 1
    }

    fmt.Fprintf(w, "query=%s tags=%v page=%d", q, tags, page)
}
```

::: warning
`strconv.Atoi("")` returns `0, err`. Without an explicit error check, pagination or filtering logic can silently become incorrect.
:::

## Working with JSON

JSON is the main data exchange format for modern APIs. [`json.Decoder`](https://pkg.go.dev/encoding/json#Decoder) and [`json.Encoder`](https://pkg.go.dev/encoding/json#Encoder) work directly with `io.Reader` and `io.Writer`, without loading the whole request or response body into memory. This differs from [`json.Unmarshal`](https://pkg.go.dev/encoding/json#Unmarshal) and [`json.Marshal`](https://pkg.go.dev/encoding/json#Marshal), which operate on byte slices.

```go
// json.Decoder reads from the stream without buffering the whole body.
if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
    http.Error(w, "Invalid JSON", http.StatusBadRequest)
    return
}
```

### Decoding a Request

```go
type CreateUserRequest struct {
    Name  string `json:"name"`
    Email string `json:"email"`
}

func createUserHandler(w http.ResponseWriter, r *http.Request) {
    mediaType, _, err := mime.ParseMediaType(r.Header.Get("Content-Type"))
    if err != nil || mediaType != "application/json" {
        http.Error(w, "Content-Type must be application/json", http.StatusUnsupportedMediaType)
        return
    }

    dec := json.NewDecoder(r.Body)
    // DisallowUnknownFields rejects JSON fields that are not declared in the struct.
    // This protects against field name typos and accidental extra data.
    dec.DisallowUnknownFields()

    var req CreateUserRequest
    if err := dec.Decode(&req); err != nil {
        http.Error(w, "Invalid JSON: "+err.Error(), http.StatusBadRequest)
        return
    }

    // json.Decoder can read multiple JSON values from one stream.
    // The first Decode read only the first object, so a second Decode checks
    // that the body does not contain another JSON document.
    if err := dec.Decode(&struct{}{}); err != io.EOF {
        http.Error(w, "Request body must contain a single JSON object", http.StatusBadRequest)
        return
    }

    // ...
}
```

### The writeJSON Helper

Setting the `Content-Type` header, writing the status code and encoding the response body are repeated in almost every JSON handler. A small helper removes that duplication:

```go
// writeJSON encodes v as JSON and sends the response with the given status code.
// After w.WriteHeader is called, the status code can no longer be changed,
// so encoding errors can only be logged.
func writeJSON(w http.ResponseWriter, status int, v any) {
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    if err := json.NewEncoder(w).Encode(v); err != nil {
        log.Printf("writeJSON: %v", err)
    }
}
```

::: info
[`json.Encoder.Encode`](https://pkg.go.dev/encoding/json#Encoder.Encode) appends `\n` to the output. In HTTP responses, this is harmless: JSON clients ignore trailing whitespace.
:::

### JSON API Error Format

Examples often use [`http.Error`](https://pkg.go.dev/net/http#Error) because it keeps error handling compact and clearly shows the early return from a handler. But `http.Error` returns `text/plain` by default, so real JSON APIs usually use a consistent JSON error format.

```go
type errorResponse struct {
    Error string `json:"error"`
}

func writeJSONError(w http.ResponseWriter, status int, message string) {
    writeJSON(w, status, errorResponse{Error: message})
}
```

This helper lets every handler return a predictable response body:

```json
{"error":"invalid JSON"}
```

In small learning examples, `http.Error` is still a reasonable simplification. In APIs where clients expect JSON, it is better to use one error format and document it as part of the API contract.

## HTML Forms and URL-encoded Data

HTML forms with `Content-Type: application/x-www-form-urlencoded` are parsed with [`r.ParseForm()`](https://pkg.go.dev/net/http#Request.ParseForm). Calling it explicitly lets you handle parsing errors. The [`r.FormValue`](https://pkg.go.dev/net/http#Request.FormValue) and [`r.PostFormValue`](https://pkg.go.dev/net/http#Request.PostFormValue) methods call `ParseForm` internally, but silently ignore any parsing error.

The right accessor depends on whether URL query parameters should be included:

| Method                                                                      | Data source |
| :-------------------------------------------------------------------------- | :---------- |
| [`r.FormValue(key)`](https://pkg.go.dev/net/http#Request.FormValue)         | Form body **and** URL query parameters. |
| [`r.PostFormValue(key)`](https://pkg.go.dev/net/http#Request.PostFormValue) | Request body only. |

```go
func formHandler(w http.ResponseWriter, r *http.Request) {
    if err := r.ParseForm(); err != nil {
        http.Error(w, "Parse error", http.StatusBadRequest)
        return
    }

    // FormValue checks both the form body and URL query parameters.
    name := r.FormValue("name")

    // PostFormValue checks only the request body and ignores query parameters.
    // Prefer it when the data source must be isolated.
    email := r.PostFormValue("email")

    fmt.Fprintf(w, "name=%s email=%s", name, email)
}
```

## File Uploads

File uploads use the `multipart/form-data` format. When handling uploads, always limit the size of incoming data and validate the actual file content by its byte signature. The file extension provided by the client is not trustworthy.

The [`multipart.FileHeader`](https://pkg.go.dev/mime/multipart#FileHeader) struct contains metadata about an uploaded file:

| Field      | Type                                                                  | Description |
| :--------- | :-------------------------------------------------------------------- | :---------- |
| `Filename` | `string`                                                              | File name provided by the client. |
| `Size`     | `int64`                                                               | File size in bytes. |
| `Header`   | [`textproto.MIMEHeader`](https://pkg.go.dev/net/textproto#MIMEHeader) | MIME part headers. |

::: warning
Do not use `Filename` as a file name on disk: it may contain path separators or special characters, leading to a [path traversal attack](https://en.wikipedia.org/wiki/Directory_traversal_attack). Store the file under a unique name generated by the server.
:::

```go
// uuid "github.com/google/uuid"

func uploadHandler(w http.ResponseWriter, r *http.Request) {
    // Limit body size to avoid exhausting memory
    // when very large files are uploaded.
    r.Body = http.MaxBytesReader(w, r.Body, 50<<20) // 50 MiB

    // FormFile returns the first file for the given key.
    file, header, err := r.FormFile("upload")
    if err != nil {
        http.Error(w, "Invalid upload", http.StatusBadRequest)
        return
    }
    defer file.Close()

    // Detect MIME type by the file byte signature: content sniffing.
    buf := make([]byte, 512)
    n, err := file.Read(buf)
    if err != nil && err != io.EOF {
        http.Error(w, "Read error", http.StatusInternalServerError)
        return
    }

    mimeType := http.DetectContentType(buf[:n])
    switch mimeType {
    case "image/jpeg", "image/png":
        // allowed types
    default:
        http.Error(w, "Unsupported file type", http.StatusUnsupportedMediaType)
        return
    }

    // After type detection, the file offset has moved up to 512 bytes.
    // Reset it before copying the whole file.
    if _, err := file.Seek(0, io.SeekStart); err != nil {
        http.Error(w, "Seek error", http.StatusInternalServerError)
        return
    }

    // header.Filename is logged for audit purposes,
    // but it is not used as the file name on disk.
    // A UUID prevents overwriting existing files and path traversal attacks.
    log.Printf("uploading file: %s (%d bytes)", header.Filename, header.Size)

    dst, err := os.Create(filepath.Join("uploads", uuid.NewString()))
    if err != nil {
        http.Error(w, "Server error", http.StatusInternalServerError)
        return
    }
    defer dst.Close()

    if _, err := io.Copy(dst, file); err != nil {
        http.Error(w, "Save error", http.StatusInternalServerError)
        return
    }
}
```

::: warning
[`r.FormFile`](https://pkg.go.dev/net/http#Request.FormFile) calls [`r.ParseMultipartForm`](https://pkg.go.dev/net/http#Request.ParseMultipartForm) with a 32 MiB in-memory limit. Data beyond that limit is stored in temporary files on disk. To control the limit explicitly, or to access all uploaded files through `r.MultipartForm`, call `r.ParseMultipartForm(maxMemory)` before using `r.FormFile`.
:::

## Limiting Request Body Size

By default, [`net/http`](https://pkg.go.dev/net/http) does not limit request body size. A single request with a multi-gigabyte body can exhaust server memory.

[`http.MaxBytesReader`](https://pkg.go.dev/net/http#MaxBytesReader) wraps `r.Body` and stops reads after the limit is exceeded. The returned error has type [`*http.MaxBytesError`](https://pkg.go.dev/net/http#MaxBytesError), which lets you distinguish it from other failures and return the correct status to the client.

```go
func restrictedHandler(w http.ResponseWriter, r *http.Request) {
    const maxBytes = 1 << 20 // 1 MiB

    // MaxBytesReader replaces r.Body: later readers such as json.Decoder or io.ReadAll
    // automatically receive an error when the limit is exceeded.
    r.Body = http.MaxBytesReader(w, r.Body, maxBytes)

    var data map[string]any
    if err := json.NewDecoder(r.Body).Decode(&data); err != nil {
        // errors.As distinguishes a size limit violation from a JSON parsing error,
        // so the handler can return a semantically correct status code.
        var maxBytesErr *http.MaxBytesError
        if errors.As(err, &maxBytesErr) {
            http.Error(w, "Request body too large", http.StatusRequestEntityTooLarge)
            return
        }
        http.Error(w, "Invalid JSON", http.StatusBadRequest)
        return
    }

    // ...
}
```
