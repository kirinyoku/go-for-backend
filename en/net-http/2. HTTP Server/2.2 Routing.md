# Routing

[`http.ServeMux`](https://pkg.go.dev/net/http#ServeMux) is the standard multiplexer, or router, in the [`net/http`](https://pkg.go.dev/net/http) package. It implements [`http.Handler`](https://pkg.go.dev/net/http#Handler) and matches incoming requests to registered handlers based on URL patterns.

Starting with Go 1.22, `http.ServeMux` became more capable: it gained built-in support for HTTP methods and named path segments, often called wildcards. This makes it possible to build fairly expressive APIs without third-party routing libraries.

## Registering Handlers

Routes are registered with [`Handle`](https://pkg.go.dev/net/http#ServeMux.Handle) and [`HandleFunc`](https://pkg.go.dev/net/http#ServeMux.HandleFunc). The difference is the type of argument they accept:

```go
mux := http.NewServeMux()

// Handle accepts a value that implements http.Handler.
mux.Handle("/api/v1/", apiHandler)

// HandleFunc accepts a function with the handler signature.
mux.HandleFunc("GET /health", func(w http.ResponseWriter, r *http.Request) {
    w.WriteHeader(http.StatusOK)
    fmt.Fprintln(w, "OK")
})
```

::: tip
For applications, use a local `http.NewServeMux()` instead of the global `http.DefaultServeMux`.
:::

## Route Patterns in Go 1.22+

A pattern can include an HTTP method, a path and special syntax for capturing path values.

### 1. HTTP Methods

The method is written at the beginning of the pattern. If it is omitted, the route matches any method. Registering `GET` also allows `HEAD` requests.

```go
mux.HandleFunc("GET /articles", listHandler)    // GET only
mux.HandleFunc("POST /articles", createHandler) // POST only
mux.HandleFunc("/public/", fileHandler)         // Any method
```

### 2. Named Segments and Wildcards

Named segments let you extract dynamic parts of the path.

- `{name}` matches one path segment.
- `{name...}` captures all remaining path segments.

```go
mux.HandleFunc("GET /users/{id}", func(w http.ResponseWriter, r *http.Request) {
    // PathValue returns the placeholder value as a string.
    idStr := r.PathValue("id")

    // Validation and type conversion are still required.
    id, err := strconv.Atoi(idStr)
    if err != nil {
        http.Error(w, "invalid user id", http.StatusBadRequest)
        return
    }

    fmt.Fprintf(w, "User ID: %d\n", id)
})
```

### 3. Exact and Prefix Matches

| Pattern     | Match type | Description |
| :---------- | :--------- | :---------- |
| `/path`     | Exact      | Matches only `/path`. |
| `/path/`    | Prefix     | Matches `/path/` and any nested path, such as `/path/sub`. |
| `/path/{$}` | Exact      | Matches only `/path/` with the trailing slash. |

## Route Precedence

`http.ServeMux` chooses a handler using the **most specific match** rule. Registration order in code does not affect the result. A pattern is more specific if it describes a narrower subset of requests.

| Criterion           | Example                    | Why it is more specific |
| :------------------ | :------------------------- | :---------------------- |
| **Method presence** | `GET /` > `/`              | Specifying a method narrows the match. |
| **Segment type**    | `/users/me` > `/users/{id}` | A literal segment is more specific than a wildcard. |
| **Path length**     | `/a/b/c` > `/a/b/`         | More path segments narrow the match. |

::: warning
If patterns overlap but neither one is more specific than the other, the mux panics to avoid ambiguity. A classic example is `GET /` and `/index.html`. The first matches all GET requests, while the second matches requests to `/index.html` with any method. For `GET /index.html`, the match is ambiguous because neither rule is strictly more specific.
:::

## Nested Routing

Because a mux is itself an `http.Handler`, routes can be grouped by nesting one `ServeMux` inside another. This is useful for API versioning or applying middleware to a specific branch.

```go
// Create an isolated route group.
api := http.NewServeMux()
api.HandleFunc("GET /users", listUsers)

root := http.NewServeMux()
// StripPrefix removes the prefix before passing the request to the nested router.
root.Handle("/api/v1/", http.StripPrefix("/api/v1", api))
```

::: warning
If a path with a trailing slash is registered, such as `/api/`, requests without that slash (`/api`) are redirected automatically with status `301`. For some clients, this can lose the request body for `POST` or `PUT` requests.
:::

## Thread Safety

`http.ServeMux` is safe for concurrent use: `Handle` and `HandleFunc` are protected by a mutex. Still, it is best to register all routes during initialization, before starting the server.
