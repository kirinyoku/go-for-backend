# Starting a Basic Server

The [`net/http`](https://pkg.go.dev/net/http) package gives you two common ways to start a basic HTTP server: the high-level [`http.ListenAndServe`](https://pkg.go.dev/net/http#ListenAndServe) function for quick setup, or an explicit [`http.Server`](https://pkg.go.dev/net/http#Server) when you need control over configuration and behavior.

## Quick Start: http.ListenAndServe

The `http.ListenAndServe` function starts a server in one line. It creates an `http.Server` with default settings and starts listening on the given address.

```go
package main

import (
    "fmt"
    "log"
    "net/http"
)

func main() {
    // Register a handler in the global DefaultServeMux.
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "Hello, World!")
    })

    // ListenAndServe blocks.
    // During normal operation, it never returns nil.
    err := http.ListenAndServe(":8080", nil)
    if err != nil {
        log.Fatalf("server failed: %s", err)
    }
}
```

The second argument to `http.ListenAndServe` is an implementation of the [`http.Handler`](https://pkg.go.dev/net/http#Handler) interface. Passing `nil` tells the server to use [`http.DefaultServeMux`](https://pkg.go.dev/net/http#DefaultServeMux), the package-level global router.

### Limitations

Using `http.ListenAndServe` together with global state is not recommended for long-lived services:

- **Global state:** `http.DefaultServeMux` can be modified by any imported package. This can expose endpoints you did not register explicitly, for example after importing [`net/http/pprof`](https://pkg.go.dev/net/http/pprof).
- **No timeouts:** timeouts are not set by default, which can leave the server vulnerable to slow clients.

## Starting a Server with http.Server

The `http.Server` struct is the main tool for building reliable services. It lets you configure timeouts, limits and TLS settings explicitly.

```go
package main

import (
    "errors"
    "log"
    "net/http"
    "time"
)

func main() {
    // Create a local router.
    mux := http.NewServeMux()
    mux.HandleFunc("GET /health", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
    })

    server := &http.Server{
        Addr: ":8080",
        // Use the local mux explicitly
        // instead of the global http.DefaultServeMux.
        Handler: mux,

        // Maximum time allowed to read request headers.
        ReadHeaderTimeout: 5 * time.Second,
        // Maximum time allowed to read the whole request: headers and body.
        ReadTimeout: 10 * time.Second,
        // Maximum time allowed to run the handler and write the response.
        // This timer starts after the request headers have been read.
        WriteTimeout: 10 * time.Second,
        // Maximum time to wait for the next request on a keep-alive connection.
        IdleTimeout: 120 * time.Second,
    }

    log.Printf("server listening on %s", server.Addr)

    // Checking for http.ErrServerClosed separates an intentional shutdown from a real error.
    if err := server.ListenAndServe(); err != nil && !errors.Is(err, http.ErrServerClosed) {
        log.Fatalf("server listen error: %s", err)
    }
}
```

::: info
`http.Server` does not have a field for limiting request body size, for example to protect against oversized uploads. That limit is usually enforced in handlers with [`http.MaxBytesReader`](https://pkg.go.dev/net/http#MaxBytesReader). This is covered in more detail in [Resource Limits](/en/net-http/server/resource-limits).
:::

### Configuration Fields

| Field               | Purpose                                                            |
| :------------------ | :----------------------------------------------------------------- |
| `Addr`              | Address to listen on (`host:port`).                                |
| `Handler`           | Main request handler, usually a router.                            |
| `ReadHeaderTimeout` | Time limit for receiving request headers. Helps avoid slow or stuck clients. |
| `ReadTimeout`       | Time limit for reading the full request.                           |
| `WriteTimeout`      | Time limit for building and sending the response.                   |
| `IdleTimeout`       | Time limit for waiting for a new request on an open connection.     |
| `MaxHeaderBytes`    | Maximum header size. The default is 1 MB.                           |
| `TLSConfig`         | Configuration for HTTPS.                                           |

## Secure Connections: HTTPS

HTTPS is enabled with the [`ListenAndServeTLS`](https://pkg.go.dev/net/http#Server.ListenAndServeTLS) method. The server configuration stays the same except for the certificate and private key paths. Detailed HTTPS setup is covered in [HTTPS](/en/net-http/server/tls-and-https).

```go
server := &http.Server{
    Addr:              ":443",
    Handler:           mux,
    ReadHeaderTimeout: 5 * time.Second,
}

// cert.pem and key.pem are the certificate and private key files.
err := server.ListenAndServeTLS("cert.pem", "key.pem")
if err != nil && !errors.Is(err, http.ErrServerClosed) {
    log.Fatalf("tls server failed: %s", err)
}
```

When TLS is enabled, `net/http` automatically configures HTTP/2 support through ALPN.
