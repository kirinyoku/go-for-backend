# Testing with Servers

The [`httptest.ResponseRecorder`](https://pkg.go.dev/net/http/httptest#ResponseRecorder) from the [Testing Handlers](./testing-handlers) article tests function logic in isolation. However, some behaviors only surface during actual network interactions:

- [`http.Client`](https://pkg.go.dev/net/http#Client) behavior — following redirects, [`CookieJar`](https://pkg.go.dev/net/http#CookieJar) operation, and timeouts.
- TLS handshakes and secure connection parameters.
- [`http.Transport`](https://pkg.go.dev/net/http#Transport) configurations — proxies, connection reuse, and protocol selection.

For these scenarios, the [`net/http/httptest`](https://pkg.go.dev/net/http/httptest) package provides tools to launch fully functional HTTP servers on random ports on the local loopback interface.

## The Test HTTP Server

[`httptest.NewServer`](https://pkg.go.dev/net/http/httptest#NewServer) starts an HTTP server and returns an [`*httptest.Server`](https://pkg.go.dev/net/http/httptest#Server). Once the test finishes, the server must be explicitly stopped by calling its [`Close`](https://pkg.go.dev/net/http/httptest#Server.Close) method.

```go
// integration_test.go
package main

import (
	"fmt"
	"io"
	"net/http"
	"net/http/httptest"
	"testing"
)

func TestGreeting(t *testing.T) {
	ts := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "Hello, client")
	}))
	// defer guarantees the server stops and the port is released even if the test panics.
	defer ts.Close()

	// ts.URL contains an address like "http://127.0.0.1:PORT".
	resp, err := http.Get(ts.URL)
	if err != nil {
		t.Fatal(err)
	}
	defer resp.Body.Close()

	body, err := io.ReadAll(resp.Body)
	if err != nil {
		t.Fatal(err)
	}

	// fmt.Fprintln appends a newline character — this is part of the expected body.
	if got, want := string(body), "Hello, client\n"; got != want {
		t.Errorf("body: got %q, want %q", got, want)
	}
}
```

## Closing the Server with t.Cleanup

For `httptest.Server`, a specific lifecycle rule applies: if a server is created within a helper function, it should be closed using `t.Cleanup` rather than `defer` inside the helper.

```go
func newTestServer(t testing.TB, handler http.Handler) *httptest.Server {
	t.Helper()
	ts := httptest.NewServer(handler)
	t.Cleanup(ts.Close)
	return ts
}
```

`t.Cleanup` will execute after the calling test completes, ensuring the server remains available throughout the entire test scenario.

```go
func TestGreeting(t *testing.T) {
	ts := newTestServer(t, http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "Hello, client")
	}))

	resp, err := http.Get(ts.URL)
	if err != nil {
		t.Fatal(err)
	}
	defer resp.Body.Close()

	body, err := io.ReadAll(resp.Body)
	if err != nil {
		t.Fatal(err)
	}

	if got, want := string(body), "Hello, client\n"; got != want {
		t.Errorf("body: got %q, want %q", got, want)
	}
}
```

## Testing TLS Connections

[`httptest.NewTLSServer`](https://pkg.go.dev/net/http/httptest#NewTLSServer) starts an HTTPS server with an automatically generated self-signed certificate. The standard [`http.DefaultClient`](https://pkg.go.dev/net/http#DefaultClient) does not trust this certificate and will return a TLS verification error.

For these tests, `*httptest.Server` provides a [`Client()`](https://pkg.go.dev/net/http/httptest#Server.Client) method, which returns an [`*http.Client`](https://pkg.go.dev/net/http#Client) preconfigured to trust this specific certificate.

```go
// newTLSTestServer starts a test HTTPS server and returns it along with
// a preconfigured client. You must use this client instead of
// http.DefaultClient — the latter would reject the self-signed certificate.
func newTLSTestServer(t testing.TB, handler http.Handler) (*httptest.Server, *http.Client) {
	t.Helper()
	ts := httptest.NewTLSServer(handler)
	t.Cleanup(ts.Close)
	return ts, ts.Client()
}

func TestSecureEndpoint(t *testing.T) {
	ts, client := newTLSTestServer(t, http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		w.WriteHeader(http.StatusOK)
	}))

	resp, err := client.Get(ts.URL)
	if err != nil {
		t.Fatal(err)
	}
	defer resp.Body.Close()

	if resp.StatusCode != http.StatusOK {
		t.Errorf("status: got %d, want %d", resp.StatusCode, http.StatusOK)
	}

	// resp.TLS contains the negotiated TLS connection parameters.
	// nil means the connection was plain, unencrypted HTTP.
	if resp.TLS == nil {
		t.Error("expected TLS connection, got plain HTTP")
	}
}
```

::: warning
`ts.Client()` is a test client tied to a specific `httptest.Server`: it trusts that server's certificate, closes its idle connections when `ts.Close()` is called, and might route requests for `example.com` to the test server. Do not use it as a general production client.
:::

## Configuring the Server Before Start

[`NewUnstartedServer`](https://pkg.go.dev/net/http/httptest#NewUnstartedServer) creates a server but doesn't start it immediately. This allows you to modify the configuration prior to startup: once [`Start()`](https://pkg.go.dev/net/http/httptest#Server.Start) or [`StartTLS()`](https://pkg.go.dev/net/http/httptest#Server.StartTLS) is called, the configuration should be considered locked — late changes might not affect the running server and could lead to data races.

A typical use case is enforcing a minimum TLS version:

```go
func TestTLSVersionEnforcement(t *testing.T) {
	ts := httptest.NewUnstartedServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		w.WriteHeader(http.StatusOK)
	}))

	// ts.TLS is set before StartTLS — do not modify the config after starting.
	ts.TLS = &tls.Config{
		MinVersion: tls.VersionTLS13,
	}
	ts.StartTLS()
	t.Cleanup(ts.Close)

	resp, err := ts.Client().Get(ts.URL)
	if err != nil {
		t.Fatal(err)
	}
	defer resp.Body.Close()

	if resp.TLS == nil {
		t.Fatal("expected TLS connection")
	}
	// resp.TLS.Version contains the numerical ID of the TLS version.
	if got, want := resp.TLS.Version, uint16(tls.VersionTLS13); got != want {
		t.Errorf("TLS version: got %#x, want %#x", got, want)
	}
}
```

To customize a non-TLS HTTP server — for instance, setting timeouts — use `ts.Start()`:

```go
ts := httptest.NewUnstartedServer(handler)
// ReadTimeout is set before starting: do not modify the config after Start().
ts.Config.ReadTimeout = 500 * time.Millisecond
ts.Start()
t.Cleanup(ts.Close)
```

## Testing Redirects and Cookies

A test server lets you verify scenarios that are impossible with a `ResponseRecorder`: how a client follows a chain of redirects and retains cookies set in responses.

```go
func TestClientRedirectsAndCookies(t *testing.T) {
	ts := newTestServer(t, http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		switch r.URL.Path {
		case "/old":
			// Permanent redirect: the client should automatically jump to /new.
			http.Redirect(w, r, "/new", http.StatusMovedPermanently)
		case "/new":
			http.SetCookie(w, &http.Cookie{Name: "session", Value: "active"})
			w.WriteHeader(http.StatusOK)
		default:
			http.NotFound(w, r)
		}
	}))

	// A CookieJar is required to automatically store and send cookies.
	// http.DefaultClient has no CookieJar — cookies would be ignored.
	jar, err := cookiejar.New(nil)
	if err != nil {
		t.Fatal(err)
	}
	client := &http.Client{Jar: jar}

	resp, err := client.Get(ts.URL + "/old")
	if err != nil {
		t.Fatal(err)
	}
	defer resp.Body.Close()

	// resp.Request contains the final request executed after all redirects.
	if got, want := resp.Request.URL.Path, "/new"; got != want {
		t.Errorf("final path: got %q, want %q", got, want)
	}

	if resp.StatusCode != http.StatusOK {
		t.Errorf("status: got %d, want %d", resp.StatusCode, http.StatusOK)
	}

	cookies := jar.Cookies(resp.Request.URL)
	if len(cookies) == 0 {
		t.Fatal("cookies: got none, want at least one")
	}
	if got, want := cookies[0].Name, "session"; got != want {
		t.Errorf("cookie name: got %q, want %q", got, want)
	}
	if got, want := cookies[0].Value, "active"; got != want {
		t.Errorf("cookie value: got %q, want %q", got, want)
	}
}
```
