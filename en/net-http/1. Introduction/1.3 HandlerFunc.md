# The http.HandlerFunc Type

Implementing [`http.Handler`](https://pkg.go.dev/net/http#Handler) with a struct requires a separate type with a [`ServeHTTP`](https://pkg.go.dev/net/http#Handler.ServeHTTP) method. For simple handlers that do not need internal state, the Go standard library provides [`http.HandlerFunc`](https://pkg.go.dev/net/http#HandlerFunc). It lets ordinary functions serve HTTP requests and removes some of the boilerplate around small handlers.

## HandlerFunc Definition

`http.HandlerFunc` is a named type based on a function signature. It implements `http.Handler` through a `ServeHTTP` method that simply calls the function itself.

```go
type HandlerFunc func(http.ResponseWriter, *http.Request)

// ServeHTTP calls f(w, r), allowing the type to satisfy the Handler interface.
func (f HandlerFunc) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    f(w, r)
}
```

## Converting a Function to http.Handler

Using `http.HandlerFunc` is a type conversion, not a function call.

```go
func healthCheck(w http.ResponseWriter, r *http.Request) {
    w.WriteHeader(http.StatusOK)
    fmt.Fprint(w, "OK")
}

// http.HandlerFunc adapts an ordinary function to the http.Handler interface.
var handler http.Handler = http.HandlerFunc(healthCheck)
```

The `healthCheck` function itself does not implement `http.Handler`. After conversion to `http.HandlerFunc`, the resulting value has a `ServeHTTP` method and becomes compatible with the interface.

## Dependency Injection

Closures let you pass external values into a handler without global variables. A constructor function can accept configuration or dependencies and return an `http.HandlerFunc`.

```go
func NewHealthHandler(version string) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Content-Type", "text/plain")

        // Response write error handling is omitted.
        fmt.Fprintf(w, "OK %s", version)
    }
}
```

::: tip
Closures work well for small handlers and middleware when there are only a few dependencies and the behavior fits naturally in one function. If the handler grows, uses several dependencies or needs a set of related methods, a struct with a `ServeHTTP` method is usually easier to maintain.
:::

## HandlerFunc in Middleware

Middleware is often described as a function that accepts an `http.Handler` and returns a new `http.Handler`. `http.HandlerFunc` is useful for these wrappers: it lets you return an ordinary function in a place where an interface is expected.

```go
func LoggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        log.Printf("%s %s", r.Method, r.URL.Path)
        next.ServeHTTP(w, r)
    })
}
```

::: info
Middleware is covered in detail in the article [Middleware](/en/net-http/server/middleware).
:::
