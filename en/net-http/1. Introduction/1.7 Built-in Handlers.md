# Built-in Handlers

The [`net/http`](https://pkg.go.dev/net/http) standard library provides ready-made components that are compatible with the [`http.Handler`](https://pkg.go.dev/net/http#Handler) interface. They are useful not only as utilities, but also as examples of the package architecture: different tasks can plug into the same `ServeHTTP` contract.

This is not an official classification of the standard library, but for an overview it is useful to group built-in handlers by role:

- **handler wrappers** accept one `http.Handler` and return another `http.Handler`, adding behavior around it;
- **ready-made responses** build the response directly and usually act as the last step in the chain.

## Handler Wrappers

Handler wrappers extend an existing handler. This style matches the general middleware model: the outer handler receives the request, performs its infrastructure work and then passes control to the inner handler.

```go
mux := http.NewServeMux()
mux.HandleFunc("POST /upload", uploadHandler)

// Limit request body reads for all routes inside mux.
handler := http.MaxBytesHandler(mux, 2<<20)
```

Common wrapper components include:

| Component                                                             | Purpose                                                                        |
| :-------------------------------------------------------------------- | :----------------------------------------------------------------------------- |
| [`http.MaxBytesHandler`](https://pkg.go.dev/net/http#MaxBytesHandler) | Limits how many bytes can be read from the request body.                       |
| [`http.TimeoutHandler`](https://pkg.go.dev/net/http#TimeoutHandler)   | Returns `503 Service Unavailable` if the inner handler does not finish in time. |
| [`http.StripPrefix`](https://pkg.go.dev/net/http#StripPrefix)         | Removes a path prefix before passing the request to the next handler.           |

## Serving Files

[`http.FileServer`](https://pkg.go.dev/net/http#FileServer) and [`http.FileServerFS`](https://pkg.go.dev/net/http#FileServerFS) serve files as ordinary HTTP responses. They are often combined with `http.StripPrefix` when the URL prefix does not match the path inside the file system.

```go
mux := http.NewServeMux()

files := http.FileServer(http.Dir("./public"))
mux.Handle("/static/", http.StripPrefix("/static/", files))
```

`http.FileServerFS` solves the same problem for file systems based on [`io/fs.FS`](https://pkg.go.dev/io/fs#FS), for example files embedded with the [`embed`](https://pkg.go.dev/embed) package.

::: warning
Do not use `FileServer` to serve the project root, configuration directories, secrets or internal files. Static assets should live in a dedicated directory such as `public` or `assets`.
:::

## Ready-made Responses

Some built-in components produce a response immediately and do not require an inner handler.

### http.RedirectHandler

[`http.RedirectHandler`](https://pkg.go.dev/net/http#RedirectHandler) returns a handler that redirects every incoming request.

```go
mux := http.NewServeMux()
mux.Handle("/old", http.RedirectHandler("/new", http.StatusMovedPermanently))
```

Temporary redirects usually use `302` or `307`; permanent redirects use `301` or `308`. Status codes `307` and `308` preserve the HTTP method, which makes them safer for API scenarios where a `POST` must not turn into a `GET`.

### http.NotFoundHandler

[`http.NotFoundHandler`](https://pkg.go.dev/net/http#NotFoundHandler) returns a handler that always responds with `404 Not Found`.

```go
mux := http.NewServeMux()
mux.Handle("/internal/", http.NotFoundHandler())
```

`ServeMux` already returns `404` for unknown routes, but `NotFoundHandler` is useful when you need to explicitly close a URL subtree or plug the standard response into more complex routing. Routing configuration is covered in detail in the article [Routing](/en/net-http/server/routing).
