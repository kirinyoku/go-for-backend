# The http.Request Struct

The [`http.Request`](https://pkg.go.dev/net/http#Request) struct contains the HTTP request data a handler works with. The same struct is also used on the client side for outgoing requests, but this article focuses mostly on incoming server requests. Creating and sending client requests is covered separately in [Creating and Sending Requests](/en/net-http/client/making-requests).

## Key Fields

The fields of `http.Request` provide access to request metadata, headers and content.

| Field           | Type                                                                    | Purpose                                                     |
| :-------------- | :---------------------------------------------------------------------- | :---------------------------------------------------------- |
| `Method`        | `string`                                                                | HTTP method (`GET`, `POST`, `PUT`, etc.)                    |
| `URL`           | [`*url.URL`](https://pkg.go.dev/net/url#URL)                            | Parsed URL with its components: path and query parameters   |
| `Proto`         | `string`                                                                | Protocol version (`"HTTP/1.1"`, `"HTTP/2"`)                 |
| `Header`        | [`http.Header`](https://pkg.go.dev/net/http#Header)                     | Request header map                                          |
| `Body`          | [`io.ReadCloser`](https://pkg.go.dev/io#ReadCloser)                     | Request body represented as a data stream                   |
| `Host`          | `string`                                                                | Value of the `Host` header or host name from the URL        |
| `ContentLength` | `int64`                                                                 | Body length in bytes (`-1` if unknown)                      |
| `RemoteAddr`    | `string`                                                                | Sender network address (`IP:port`), set by the server       |
| `TLS`           | [`*tls.ConnectionState`](https://pkg.go.dev/crypto/tls#ConnectionState) | TLS connection state for HTTPS requests                     |

## Working with the Request Body

The `Body` field has type `io.ReadCloser` and represents the request body as a stream. This streaming model has two important consequences:

1. **Single read:** after the stream has been fully consumed, it is exhausted. Further reads return [`io.EOF`](https://pkg.go.dev/io#EOF).
2. **Memory efficiency:** streaming allows the server to process large bodies without loading the whole payload into memory.

::: info
On the server side, `net/http` provides a non-nil `Body`; for requests without a body, reading from it returns `io.EOF` immediately. On the client side, when creating a request, `Body` may be `nil` if no body was provided.
:::

### Reading the Body More Than Once

To reuse a request body, for example when logging it in middleware, you need to save the data and replace `Body` with a new stream. This is a low-level technique, not the normal way request data is usually handled. JSON, forms, multipart uploads and size limits are covered in detail in the article [Reading and Writing Data](/en/net-http/server/working-with-data).

```go
func loggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // Keep a reference to the original network stream.
        originalBody := r.Body

        // Defer closing the original stream.
        // This call will still close the original stream even after r.Body is replaced.
        defer originalBody.Close()

        // Read the body into memory.
        bodyBytes, err := io.ReadAll(originalBody)
        if err != nil {
            http.Error(w, "Internal Server Error", http.StatusInternalServerError)
            return
        }

        log.Printf("request body: %s", bodyBytes)

        // Create a new stream from the saved bytes.
        // io.NopCloser adds a no-op Close method,
        // making the reader compatible with io.ReadCloser.
        r.Body = io.NopCloser(bytes.NewReader(bodyBytes))

        next.ServeHTTP(w, r)
    })
}
```

::: warning
In real middleware, this pattern requires a limit on how much data can be read. Otherwise, logging a large request body can itself become a source of excessive memory usage.
:::

::: info
On the server side, `r.Body` is closed automatically after the handler returns.
:::

## Headers and Cookies

The `http.Header` type is a `map[string][]string`. Its methods canonicalize header keys to the `Canonical-Key-Name` form when reading and writing headers.

### Managing Headers

The [`Get()`](https://pkg.go.dev/net/http#Header.Get), [`Set()`](https://pkg.go.dev/net/http#Header.Set) and [`Add()`](https://pkg.go.dev/net/http#Header.Add) methods handle casing transparently and canonicalize keys:

```go
r.Header.Set("auth-token", "secret")
token := r.Header.Get("Auth-Token") // Returns "secret"
```

### Working with Cookies

Requests provide specialized [`r.Cookie`](https://pkg.go.dev/net/http#Request.Cookie) and [`r.Cookies`](https://pkg.go.dev/net/http#Request.Cookies) methods, so you usually do not need to parse the `Cookie` header manually:

```go
// Get a cookie by name.
cookie, err := r.Cookie("session_id")
if err != nil {
    // Handle missing cookie.
}

// Get all request cookies as a slice.
cookies := r.Cookies()
```

## Context and Copying

Every `http.Request` carries a [`context.Context`](https://pkg.go.dev/context#Context), available through [`r.Context()`](https://pkg.go.dev/net/http#Request.Context). This allows request lifecycle management and lets application layers pass request-scoped values.

### Copying a Request

Because `http.Request` is a struct rather than an interface, changes are usually made on a copy:

- **[`WithContext(ctx)`](https://pkg.go.dev/net/http#Request.WithContext)**: creates a shallow copy of the request with a new context. Reference fields such as `Header` and `Body` are shared.
- **[`Clone(ctx)`](https://pkg.go.dev/net/http#Request.Clone)**: makes a deep copy of most reference fields, including a copy of the headers. This is a safer way to modify request metadata before passing the request further down the chain.

::: warning
`Clone` does not create an independent copy of the `Body` stream. The original request and its clone point to the same request body, so reading from one exhausts the stream for the other as well. If the body must be read again, it needs to be buffered explicitly and replaced with a new `io.ReadCloser`.
:::

```go
type contextKey string
const requestIDKey contextKey = "request_id"

// Modify the request with Clone before passing it to the next handler.
ctx := context.WithValue(r.Context(), requestIDKey, "uuid-123")
newReq := r.Clone(ctx)
newReq.Header.Set("X-Request-ID", "uuid-123")

next.ServeHTTP(w, newReq)
```

::: tip
In real applications, avoid using built-in types such as `string` as context keys. This helps prevent key collisions between different packages or parts of the application. Define your own unexported key types instead.
:::
