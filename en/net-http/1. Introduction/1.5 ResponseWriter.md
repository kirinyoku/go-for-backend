# The http.ResponseWriter Interface

The [`http.ResponseWriter`](https://pkg.go.dev/net/http#ResponseWriter) interface is the main way to write an HTTP response from a server handler. A handler uses it to set headers, choose a status code and send the response body.

`ResponseWriter` does not store a completed response as a data structure. It is a writer interface that is valid only while the handler is running. A completed response is represented by a different type, [`http.Response`](/en/net-http/intro/response), which appears more often in HTTP client code and tests.

## Core Methods

Unlike [`http.Request`](https://pkg.go.dev/net/http#Request), which is a struct, `ResponseWriter` is an interface. This lets [`net/http`](https://pkg.go.dev/net/http) use different internal implementations depending on the HTTP protocol version and server configuration.

```go
type ResponseWriter interface {
    Header() Header
    WriteHeader(statusCode int)
    Write([]byte) (int, error)
}
```

| Method                                                                       | Description                                                 |
| :--------------------------------------------------------------------------- | :---------------------------------------------------------- |
| [`Header() Header`](https://pkg.go.dev/net/http#ResponseWriter.Header)       | Returns the header map that can be modified before sending. |
| [`WriteHeader(int)`](https://pkg.go.dev/net/http#ResponseWriter.WriteHeader) | Sends the HTTP status code to the client.                   |
| [`Write([]byte)`](https://pkg.go.dev/net/http#ResponseWriter.Write)          | Writes response body data and commits the headers.          |

## Operation Order

In HTTP, response data is sent in a strict order: status and headers first, then the response body. The Go standard library enforces this order automatically, but it also creates important constraints for handler code.

```mermaid
flowchart LR
    A[Header.Set] --> B[WriteHeader]
    B --> C[Write]
    C --> D[Done]
```

### 1. Modifying Headers

The `Header()` method returns an [`http.Header`](https://pkg.go.dev/net/http#Header): the header map that will be sent to the client. All header changes must be made before the first call to `Write` or `WriteHeader`.

```go
func handler(w http.ResponseWriter, r *http.Request) {
    // Set headers before writing the status or body.
    w.Header().Set("Content-Type", "application/json")

    // ... handler logic
}
```

### 2. Setting the Status

The `WriteHeader` method commits the response status code.

- If it is not called explicitly, the first write to the body sends `200 OK`.
- After `WriteHeader` is called, regular HTTP headers are considered sent to the client, so later changes to them do not affect the response.

### 3. Writing the Body

The `Write` method sends data to the client.

- If the status has not been sent yet, the first call to `Write` implicitly calls `WriteHeader(http.StatusOK)`.
- If `Content-Type` was not set manually, the library inspects the first 512 bytes of data to detect the content type automatically. This is called content sniffing.

```go
func handler(w http.ResponseWriter, r *http.Request) {
    // 1. Headers.
    w.Header().Set("Content-Type", "text/plain")
    // 2. Status code.
    w.WriteHeader(http.StatusAccepted)

    // 3. Response body.
    if _, err := w.Write([]byte("Request accepted")); err != nil {
        log.Printf("failed to write response: %v", err)
    }
}
```

::: warning
Changing headers after calling `Write` is a logic error: the server ignores those changes because the HTTP headers have already been sent to the network stack. Calling `WriteHeader` more than once also does not change the response status, and the server logs a warning: `http: superfluous response.WriteHeader call`.
:::

## Common Mistakes

Calling [`http.Error`](https://pkg.go.dev/net/http#Error), `WriteHeader` or `Write` commits the response, but it does not stop the function. If that branch of the handler is finished, return explicitly. Otherwise, the code below continues running and may try to write a different status or body.

```go
func handler(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "method not allowed", http.StatusMethodNotAllowed)
        return
    }

    fmt.Fprint(w, "created")
}
```

It is also important not to store `ResponseWriter` in global variables or use it after the handler returns. It is tied to one request and the lifecycle of its connection.

## Advanced Capabilities

The `http.ResponseWriter` interface is intentionally minimal. For streaming, deadline management and lower-level operations, Go 1.20+ provides [`http.ResponseController`](https://pkg.go.dev/net/http#ResponseController). It can call operations such as `Flush`, `SetReadDeadline`, `SetWriteDeadline` and `Hijack` when the current response implementation supports them.

For ordinary JSON/API handlers, the basic `Header`, `WriteHeader` and `Write` methods are enough. Advanced operations become important later: when you need to flush a streaming response, control read and write deadlines, take over the connection with `Hijack` or preserve these capabilities after wrapping `ResponseWriter` in middleware. These scenarios are covered later in the article [Streaming Responses](/en/net-http/server/responsecontroller).
