# Собственный обработчик

Интерфейс [`slog.Handler`](https://pkg.go.dev/log/slog#Handler) — низкоуровневая точка расширения [`log/slog`](https://pkg.go.dev/log/slog). Собственный обработчик нужен, когда запись требуется представить в новом формате, сохранить в специализированном хранилище или передать API, для которого нет готовой реализации.

Если задача сводится к JSON, тексту, изменению служебных полей или фильтрации уровня, лучше оставить эту работу встроенному обработчику и его [`HandlerOptions`](/ru/log-slog/advanced/handler-configuration). Полная реализация `Handler` должна самостоятельно соблюдать правила атрибутов, групп, производных логгеров и конкурентных вызовов.

В качестве примера будет `MemoryHandler`, сохраняющий события как `map[string]any`. Он не предназначен для рабочего хранилища логов, зато позволяет рассмотреть контракт без кода сетевого клиента или нового формата сериализации.

## Четыре метода Handler

Интерфейс состоит из четырёх методов:

```go
type Handler interface {
    Enabled(context.Context, slog.Level) bool
    Handle(context.Context, slog.Record) error
    WithAttrs([]slog.Attr) slog.Handler
    WithGroup(string) slog.Handler
}
```

Каждый метод отвечает за отдельную часть поведения:

| Метод | Ответственность |
| :--- | :--- |
| `Enabled` | Решить, нужно ли создавать и обрабатывать событие выбранного уровня. |
| `Handle` | Обработать готовый `slog.Record`. |
| `WithAttrs` | Создать производный обработчик с постоянными атрибутами. |
| `WithGroup` | Создать производный обработчик с открытой группой. |

Методы `WithAttrs` и `WithGroup` не должны изменять исходный обработчик. Базовый и производные логгеры могут использоваться одновременно, поэтому конфигурацию удобнее хранить как неизменяемые снимки, а изменяемое состояние приёмника — отдельно.

## Состояние MemoryHandler

Обработчик хранит минимальный уровень, постоянные атрибуты, путь открытых групп и общий список событий:

```go
type MemoryHandler struct {
    state  *memoryState
    level  slog.Leveler
    attrs  map[string]any
    groups []string
}

type memoryState struct {
    mu     sync.Mutex
    events []map[string]any
}

func NewMemoryHandler(level slog.Leveler) *MemoryHandler {
    if level == nil {
        level = slog.LevelInfo
    }

    return &MemoryHandler{
        state: &memoryState{},
        level: level,
        attrs: make(map[string]any),
    }
}
```

Все производные экземпляры будут ссылаться на один `memoryState`, потому что представляют один приёмник событий. Поля `attrs` и `groups` после создания экземпляра не изменяются: методы производного обработчика создадут новые значения.

Параметр типа `slog.Leveler` позволяет передать как постоянный `slog.Level`, так и динамический `*slog.LevelVar`. Значение `nil` трактуется так же, как во встроенных обработчиках: минимальным уровнем становится `Info`.

## Ранняя фильтрация

Метод `Enabled` сравнивает уровень события с текущим порогом:

```go
func (h *MemoryHandler) Enabled(
    _ context.Context,
    level slog.Level,
) bool {
    return level >= h.level.Level()
}
```

`Logger` вызывает этот метод до создания полной записи. Если он вернул `false`, метод `Handle` вызван не будет. Поэтому `Enabled` должен быть быстрым и не выполнять операции ввода-вывода.

Контекст можно учитывать при фильтрации, но это создаёт политику, зависящую от конкретного запроса. Для обычного порога он не нужен.

## Обработка Record

Метод `Handle` получает уже сформированный [`slog.Record`](https://pkg.go.dev/log/slog#Record). `MemoryHandler` создаёт карту события, переносит служебные поля и обходит атрибуты через [`Record.Attrs`](https://pkg.go.dev/log/slog#Record.Attrs):

```go
func (h *MemoryHandler) Handle(
    _ context.Context,
    record slog.Record,
) error {
    event := cloneMap(h.attrs)

    record.Attrs(func(attr slog.Attr) bool {
        addAttrAtGroups(event, h.groups, attr)
        return true
    })

    if !record.Time.IsZero() {
        event[slog.TimeKey] = record.Time
    }
    event[slog.LevelKey] = record.Level
    event[slog.MessageKey] = record.Message

    h.state.mu.Lock()
    h.state.events = append(h.state.events, event)
    h.state.mu.Unlock()

    return nil
}
```

Нулевое значение `record.Time` не добавляется в результат — это одно из требований контракта. Уровень и сообщение сохраняются под стандартными ключами, чтобы структуру события могли понимать обычные инструменты `slog`, включая `testing/slogtest`.

Поле `record.PC` содержит позицию вызова, но обработчик не обязан выводить source. `MemoryHandler` его не использует; реализация с поддержкой источника должна игнорировать нулевой `PC` и преобразовывать ненулевое значение через API `runtime` или `Record.Source` в доступных версиях Go.

Контекст передаётся в `Handle` для чтения значений операции. Отмена `ctx` не должна сама по себе прерывать обработку: запись о тайм-ауте часто создаётся уже после отмены запроса.

Поскольку событие представлено картой, повторный ключ заменяет предыдущее значение. Для этого учебного обработчика такое правило допустимо, а служебные ключи резервируются за `time`, `level` и `msg`. Реализация для собственного формата должна выбрать политику дубликатов явно; если порядок и повторения важны, вместо map понадобится последовательность атрибутов.

## Значения и группы

Атрибут может содержать обычное значение, реализацию `slog.LogValuer` или вложенную группу. Перед чтением значения обработчик вызывает [`Value.Resolve`](https://pkg.go.dev/log/slog#Value.Resolve):

```go
func addAttr(
    target map[string]any,
    attr slog.Attr,
) {
    attr.Value = attr.Value.Resolve()
    if attr.Equal(slog.Attr{}) {
        return
    }

    if attr.Value.Kind() == slog.KindGroup {
        fields := make(map[string]any)
        for _, child := range attr.Value.Group() {
            addAttr(fields, child)
        }

        if len(fields) == 0 {
            return
        }
        if attr.Key == "" {
            mergeMap(target, fields)
            return
        }

        target[attr.Key] = fields
        return
    }

    target[attr.Key] = valueAsAny(attr.Value)
}
```

`Resolve` вызывает `LogValue` столько раз, сколько нужно для получения обычного значения, и ограничивает бесконечные цепочки. Прямой одиночный вызов `LogValue` не обеспечивает такого поведения.

Нулевой `slog.Attr{}` игнорируется. Группа с пустым ключом встраивается в текущую область, а группа без полей не создаёт пустой объект. Для вложенных групп функция рекурсивно применяет те же правила к каждому дочернему атрибуту.

Известные виды `slog.Value` можно преобразовать через `Value.Any`. Для неизвестного будущего вида безопаснее оставить строковое представление, чем завершить приложение паникой:

```go
func valueAsAny(value slog.Value) any {
    switch value.Kind() {
    case slog.KindAny,
        slog.KindBool,
        slog.KindDuration,
        slog.KindFloat64,
        slog.KindInt64,
        slog.KindString,
        slog.KindTime,
        slog.KindUint64:
        return value.Any()
    default:
        return value.String()
    }
}
```

Остальные вспомогательные функции создают путь открытых групп и копируют карты перед изменением:

```go
func addAttrAtGroups(
    event map[string]any,
    groups []string,
    attr slog.Attr,
) {
    fields := make(map[string]any)
    addAttr(fields, attr)
    if len(fields) == 0 {
        return
    }

    target := groupTarget(event, groups)
    mergeMap(target, fields)
}

func groupTarget(
    event map[string]any,
    groups []string,
) map[string]any {
    target := event
    for _, name := range groups {
        child, ok := target[name].(map[string]any)
        if !ok {
            child = make(map[string]any)
            target[name] = child
        }
        target = child
    }
    return target
}

func mergeMap(
    target map[string]any,
    source map[string]any,
) {
    for key, value := range source {
        target[key] = value
    }
}

func cloneMap(source map[string]any) map[string]any {
    clone := make(map[string]any, len(source))
    for key, value := range source {
        if group, ok := value.(map[string]any); ok {
            value = cloneMap(group)
        }
        clone[key] = value
    }
    return clone
}
```

Путь групп создаётся только после появления непустого атрибута. Благодаря этому `logger.WithGroup("http").Info("request started")` не оставляет в событии пустой объект `http`.

`cloneMap` копирует созданные обработчиком вложенные карты, но не пытается копировать произвольные значения из `slog.Any`. Если атрибут содержит указатель, map или slice, прикладной код не должен изменять соответствующие данные, пока обработчик может ими пользоваться.

## Постоянные атрибуты и группы

Метод `WithAttrs` создаёт обработчик с копией накопленных полей и добавляет новые атрибуты в текущую область групп:

```go
func (h *MemoryHandler) WithAttrs(
    attrs []slog.Attr,
) slog.Handler {
    next := &MemoryHandler{
        state:  h.state,
        level:  h.level,
        attrs:  cloneMap(h.attrs),
        groups: h.groups,
    }

    for _, attr := range attrs {
        addAttrAtGroups(next.attrs, next.groups, attr)
    }

    return next
}
```

`slog.Handler` получает владение срезом `attrs` и имеет право сохранить его, но эта реализация сразу преобразует элементы в собственную карту. Исходный `MemoryHandler` остаётся неизменным.

`WithGroup` копирует путь и добавляет к нему новое имя:

```go
func (h *MemoryHandler) WithGroup(
    name string,
) slog.Handler {
    if name == "" {
        return h
    }

    groups := make(
        []string,
        len(h.groups),
        len(h.groups)+1,
    )
    copy(groups, h.groups)
    groups = append(groups, name)

    return &MemoryHandler{
        state:  h.state,
        level:  h.level,
        attrs:  h.attrs,
        groups: groups,
    }
}
```

Отдельное копирование сохраняет порядок цепочки. Например, после `logger.With("service", "users").WithGroup("http").With("method", "GET")` поле `service` остаётся в корне, а `method` помещается в группу `http`.

Пустое имя группы не влияет на квалификацию последующих полей, поэтому метод возвращает текущий обработчик без создания копии.

## Чтение сохранённых событий

Для чтения обработчик возвращает снимок под той же блокировкой, которая защищает запись:

```go
func (h *MemoryHandler) Events() []map[string]any {
    h.state.mu.Lock()
    defer h.state.mu.Unlock()

    events := make([]map[string]any, len(h.state.events))
    for i, event := range h.state.events {
        events[i] = cloneMap(event)
    }
    return events
}
```

Возвращать внутренний срез напрямую нельзя: вызывающий код смог бы изменить его одновременно с новым вызовом `Handle`. Копия отделяет снимок от дальнейшей работы логгера.

Общий `*memoryState` и его mutex важны для производных обработчиков. Если каждый результат `WithAttrs` создаст собственную блокировку, разные логгеры всё равно смогут одновременно изменить один и тот же список событий.

## Когда нужен Record.Clone

`slog.Record` передаётся в `Handle` по значению, но его копии могут разделять внутреннее хранилище атрибутов. Простого присваивания недостаточно, если запись будет изменяться или сохранится после возврата из метода:

```go
copy := record // внутреннее состояние всё ещё может быть общим
```

Перед изменением или передачей записи асинхронному потребителю используют [`Record.Clone`](https://pkg.go.dev/log/slog#Record.Clone):

```go
record = record.Clone()
record.AddAttrs(
    slog.String("transport", "queue"),
)

h.records <- record
```

`MemoryHandler` не хранит сам `Record`: он синхронно переносит атрибуты в собственную карту, поэтому вызов `Clone` в его `Handle` не нужен. Клонирование также не создаёт глубоких копий произвольных объектов, переданных через `slog.Any`.

## Конкурентность

Контракт разрешает вызывать любой метод одного обработчика одновременно с другими его методами. В `MemoryHandler` это обеспечивается двумя решениями: конфигурация экземпляра остаётся неизменяемой, а общий список событий защищён mutex.

Формирование карты выполняется без блокировки, потому что используются только локальные данные и неизменяемая конфигурация. Критическая секция охватывает лишь добавление готового события в срез. Для обработчика, записывающего в `io.Writer`, аналогичный подход означает сначала сформировать запись в локальном буфере, а затем одним защищённым вызовом записать её в общий поток.

Конкурентность нужно дополнительно проверять детектором гонок:

```bash
go test -race ./...
```

Обычный успешный тест не обнаруживает доступ к общей памяти без синхронизации.

## Проверка через testing/slogtest

Пакет [`testing/slogtest`](https://pkg.go.dev/testing/slogtest) проверяет общий контракт обработчика: служебные поля, `WithAttrs`, `WithGroup`, пустые и вложенные группы, нулевые значения и разрешение `LogValuer`.

Метод `Events` уже возвращает представление, которое ожидает [`slogtest.TestHandler`](https://pkg.go.dev/testing/slogtest#TestHandler), поэтому проверка получается короткой:

```go
func TestMemoryHandlerContract(t *testing.T) {
    handler := NewMemoryHandler(slog.LevelInfo)

    if err := slogtest.TestHandler(
        handler,
        handler.Events,
    ); err != nil {
        t.Fatal(err)
    }
}
```

`TestHandler` устанавливает обработчик в `slog.Logger`, создаёт набор событий и сопоставляет результат с требованиями интерфейса. Для обработчика, выводящего JSON, функция результатов обычно декодирует каждую строку в `map[string]any`.

Эта проверка не знает прикладного назначения реализации. Отдельные тесты всё равно нужны для собственного порога, ошибок приёмника, выбранного представления source и других гарантий. `slogtest` также не заменяет запуск с `-race`.

`MemoryHandler.Handle` всегда возвращает `nil`, потому что запись в память не имеет отдельного канала ошибок. Реальный конечный обработчик должен вернуть ошибку своего приёмника, но методы `Logger.Info`, `Logger.Error` и остальные не передадут её прикладному коду. Наблюдение за такими сбоями и резервный путь организуют внутри логирующей инфраструктуры, не пытаясь записать ошибку через тот же неисправный обработчик.

Собственный конечный обработчик оправдан, когда приложению действительно нужен новый приёмник или формат. Для добавления полей, маршрутизации и повторного использования готового JSON-кодирования обычно достаточно [обёрнуть существующий обработчик](/ru/log-slog/advanced/wrappers-and-composition).
