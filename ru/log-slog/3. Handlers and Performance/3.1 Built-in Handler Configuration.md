# Настройка встроенных обработчиков

Встроенные [`slog.TextHandler`](https://pkg.go.dev/log/slog#TextHandler) и [`slog.JSONHandler`](https://pkg.go.dev/log/slog#JSONHandler) отвечают не только за представление записи. Через [`slog.HandlerOptions`](https://pkg.go.dev/log/slog#HandlerOptions) они могут добавлять место вызова, фильтровать события и преобразовывать атрибуты перед выводом.

Эти настройки задаются при создании обработчика и применяются ко всем проходящим через него событиям. Прикладной код продолжает пользоваться обычным `*slog.Logger` и не зависит от имён служебных полей или представления итоговой записи.

## Возможности HandlerOptions

Структура `slog.HandlerOptions` содержит три поля:

| Поле | Назначение |
| :--- | :--- |
| `AddSource` | Добавляет функцию, файл и строку, из которой был вызван логгер. |
| `Level` | Задаёт минимальный разрешённый уровень. |
| `ReplaceAttr` | Преобразует или удаляет атрибут перед выводом. |

Нулевое значение `HandlerOptions` соответствует стандартному поведению: источник не добавляется, минимальным уровнем считается `Info`, а атрибуты выводятся без дополнительного преобразования. Передача `nil` в `NewJSONHandler` или `NewTextHandler` даёт тот же набор настроек.

Уровень и его динамическое изменение через `slog.LevelVar` уже разобраны в статье [«Уровни и фильтрация»](/ru/log-slog/intro/levels-and-filtering). Здесь основное внимание уделено `AddSource` и `ReplaceAttr`.

## Место вызова

Опция `AddSource` добавляет к каждой записи атрибут `source`:

```go
handler := slog.NewJSONHandler(
    os.Stdout,
    &slog.HandlerOptions{
        AddSource: true,
    },
)
logger := slog.New(handler)

logger.Error("load configuration failed")
```

JSON-представление будет выглядеть примерно так:

```json
{"time":"2026-07-22T12:00:00Z","level":"ERROR","source":{"function":"main.main","file":"/app/cmd/service/main.go","line":24},"msg":"load configuration failed"}
```

Тип [`slog.Source`](https://pkg.go.dev/log/slog#Source) содержит имя функции, путь к файлу и номер строки. Эти данные помогают перейти от события к месту в коде, но увеличивают размер каждой записи. Кроме того, полный путь к исходному файлу зависит от среды сборки и обычно не подходит для группировки событий.

Источник указывает на непосредственный вызов метода логгера. Если приложение скрывает `logger.Info` или `logger.Error` за собственной функцией-обёрткой, в `source` может оказаться строка внутри этой обёртки, а не место, из которого она была вызвана. Сохранение исходного вызывающего кода в таких API требует отдельной работы с `slog.Record` и выходит за пределы настройки встроенного обработчика.

`AddSource` действует для всех разрешённых уровней. Если место вызова нужно только отдельным событиям, одного флага недостаточно: потребуется другая схема атрибутов или собственный обработчик.

## Служебные атрибуты

Встроенный обработчик добавляет несколько атрибутов независимо от полей прикладного события:

| Константа | Стандартный ключ | Значение |
| :--- | :--- | :--- |
| [`slog.TimeKey`](https://pkg.go.dev/log/slog#TimeKey) | `time` | Время создания записи. |
| [`slog.LevelKey`](https://pkg.go.dev/log/slog#LevelKey) | `level` | Уровень события. |
| [`slog.MessageKey`](https://pkg.go.dev/log/slog#MessageKey) | `msg` | Сообщение. |
| [`slog.SourceKey`](https://pkg.go.dev/log/slog#SourceKey) | `source` | Место вызова при включённом `AddSource`. |

Константы нужны вместо строковых литералов, когда конфигурация должна распознать служебный атрибут. Например, внешняя система может ожидать ключи `@timestamp`, `severity` и `message` вместо стандартных имён `slog`.

Прикладной схеме не следует повторно использовать эти ключи на верхнем уровне. `JSONHandler` не устраняет дубликаты, а разные потребители JSON могут обрабатывать одинаковые ключи по-разному.

## Преобразование через ReplaceAttr

Поле `ReplaceAttr` принимает функцию, которая вызывается перед выводом каждого негруппового атрибута:

```go
func replaceAttr(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    return attr
}
```

В `attr` приходит уже разрешённое значение: если исходный тип реализует `slog.LogValuer`, обработчик сначала вызовет его `LogValue`. Функция может вернуть изменённый атрибут, оставить его без изменений или вернуть нулевой `slog.Attr`, чтобы удалить поле из результата.

Аргумент `groups` содержит путь открытых групп, внутри которых находится атрибут. Для служебных полей верхнего уровня этот срез пуст. Его нельзя изменять или сохранять для использования после возврата из функции.

### Переименование служебных полей

Проверка `len(groups) == 0` не позволяет случайно переименовать одноимённое поле внутри прикладной группы:

```go
func renameBuiltinAttrs(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) != 0 {
        return attr
    }

    switch attr.Key {
    case slog.TimeKey:
        attr.Key = "@timestamp"
    case slog.LevelKey:
        attr.Key = "severity"
    case slog.MessageKey:
        attr.Key = "message"
    case slog.SourceKey:
        attr.Key = "caller"
    }

    return attr
}
```

Эту функцию передают обоим встроенным обработчикам одинаково:

```go
options := &slog.HandlerOptions{
    AddSource:  true,
    ReplaceAttr: renameBuiltinAttrs,
}

handler := slog.NewJSONHandler(os.Stdout, options)
logger := slog.New(handler)
```

Теперь вызовы приложения остаются прежними, а внешний контракт записи использует выбранные имена:

```json
{"@timestamp":"2026-07-22T12:00:00Z","severity":"INFO","caller":{"function":"main.main","file":"/app/cmd/service/main.go","line":31},"message":"service started"}
```

### Преобразование значений

`ReplaceAttr` может менять не только ключ, но и тип значения. Следующая функция переводит служебное время в число миллисекунд Unix:

```go
func replaceTime(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 0 && attr.Key == slog.TimeKey {
        attr.Value = slog.Int64Value(
            attr.Value.Time().UnixMilli(),
        )
    }

    return attr
}
```

Такое преобразование оправдано, когда формат является частью заранее определённого контракта с получателем логов. Менять время только ради более короткой записи обычно не нужно: стандартное строковое представление сразу сохраняет часовой пояс и читается человеком.

Для служебного `time` заранее известен тип `time.Time`. При преобразовании прикладных атрибутов сначала нужно проверить [`slog.Value.Kind`](https://pkg.go.dev/log/slog#Value.Kind), иначе метод вроде `Value.Time` может вызвать панику для значения другого типа.

### Работа с группами

`ReplaceAttr` не вызывается для группового атрибута целиком, но вызывается для каждого поля внутри него. Для такого события:

```go
logger.Info("upstream request completed",
    slog.Group("http",
        slog.Int("status", 200),
        slog.Group("upstream",
            slog.String("host", "api.example.com"),
        ),
    ),
)
```

обработчик передаст в `ReplaceAttr` следующие пути:

| Атрибут | `groups` |
| :--- | :--- |
| Служебные `time`, `level`, `msg` | пустой срез |
| `status` | `[]string{"http"}` |
| `host` | `[]string{"http", "upstream"}` |

Благодаря пути одно имя можно преобразовать только в нужном пространстве:

```go
func renameHTTPStatus(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 1 &&
        groups[0] == "http" &&
        attr.Key == "status" {
        attr.Key = "status_code"
    }

    return attr
}
```

Сравнивать только `attr.Key` недостаточно, если одинаковое имя встречается, например, в группах `http` и `database`, но имеет в них разный смысл.

### Удаление атрибута

Нулевой атрибут исключается из результата. Например, так можно убрать стандартное время:

```go
func removeTime(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 0 && attr.Key == slog.TimeKey {
        return slog.Attr{}
    }

    return attr
}
```

Удалять временную метку стоит только тогда, когда надёжное время добавляется другим слоем системы сбора. Делать это ради тестов не нужно: как показано в разделе [«Проверка схемы события»](/ru/log-slog/intro/attributes-and-event-schema#проверка-схемы-события), тест может декодировать JSON и просто не включать поле `time` в проверяемую структуру.

## Защитное преобразование полей

`ReplaceAttr` можно использовать как дополнительный барьер для атрибутов с запрещёнными именами:

```go
func redactSensitiveAttr(
    _ []string,
    attr slog.Attr,
) slog.Attr {
    switch strings.ToLower(attr.Key) {
    case "authorization", "password", "token":
        return slog.String(attr.Key, "[REDACTED]")
    default:
        return attr
    }
}
```

Такая функция увидит поля внутри `slog.Group`, потому что обработчик передаёт их по отдельности. Но значение, добавленное как `slog.Any("request", request)`, остаётся одним атрибутом: `ReplaceAttr` не обходит произвольную структуру и не сможет найти вложенное поле `Password`. Она также не удалит секрет, уже помещённый в сообщение или текст ошибки.

Поэтому преобразование является последней защитой, а не разрешением передавать в лог необработанные структуры. Основное правило остаётся прежним: код события формирует явный список безопасных полей, а типы с собственным представлением используют `LogValuer`, как разобрано в статье [«Пользовательские значения»](/ru/log-slog/events/custom-values).

## Общая конфигурация

Несколько правил можно объединить в одной функции и применить в месте создания логгера:

```go
func replaceLogAttr(
    groups []string,
    attr slog.Attr,
) slog.Attr {
    if len(groups) == 0 {
        switch attr.Key {
        case slog.TimeKey:
            attr.Key = "@timestamp"
        case slog.LevelKey:
            attr.Key = "severity"
        case slog.MessageKey:
            attr.Key = "message"
        case slog.SourceKey:
            attr.Key = "caller"

            source, ok := attr.Value.Any().(*slog.Source)
            if ok {
                source.File = filepath.Base(source.File)
            }
        }
    }

    switch strings.ToLower(attr.Key) {
    case "authorization", "password", "token":
        return slog.String(attr.Key, "[REDACTED]")
    }

    return attr
}

func newLogger(
    out io.Writer,
    level slog.Leveler,
) *slog.Logger {
    handler := slog.NewJSONHandler(
        out,
        &slog.HandlerOptions{
            AddSource:  true,
            Level:      level,
            ReplaceAttr: replaceLogAttr,
        },
    )

    return slog.New(handler)
}
```

Путь к исходному файлу сокращается через [`filepath.Base`](https://pkg.go.dev/path/filepath#Base), а имя функции и номер строки сохраняются. Параметр `slog.Leveler` принимает как постоянный `slog.Level`, так и `*slog.LevelVar`, поэтому решение о динамической фильтрации не проникает в прикладные компоненты.

`ReplaceAttr` выполняется при обработке записей и может вызываться одновременно из нескольких горутин. Функция должна быть быстрой и безопасной для конкурентного использования: не изменять общую карту без синхронизации, не обращаться к сети и не выполнять тяжёлую работу для каждого поля.

Встроенной настройки достаточно, пока преобразование зависит от отдельного атрибута и пути его групп. Для анализа всей записи, чтения значений из `context.Context`, отправки событий в разные потоки или поведения, зависящего от сочетания полей, потребуется [обёртка над готовым обработчиком](/ru/log-slog/advanced/wrappers-and-composition) либо [собственный `slog.Handler`](/ru/log-slog/advanced/custom-handler).
