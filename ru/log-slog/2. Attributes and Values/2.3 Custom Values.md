# Пользовательские значения

Набор функций [`slog.String`](https://pkg.go.dev/log/slog#String), [`slog.Int`](https://pkg.go.dev/log/slog#Int), [`slog.Bool`](https://pkg.go.dev/log/slog#Bool) и других покрывает основные типы Go. Для структуры, среза, ошибки или другого произвольного значения пакет [`log/slog`](https://pkg.go.dev/log/slog) предоставляет [`slog.Any`](https://pkg.go.dev/log/slog#Any). Если стандартного представления недостаточно, собственный тип может реализовать [`slog.LogValuer`](https://pkg.go.dev/log/slog#LogValuer) и сам определить, какие данные попадут в запись.

Такой подход позволяет сохранить структуру прикладного значения, скрыть поля, которые не должны появляться в логах, и отложить вычисление значения до момента, когда событие действительно будет обработано.

## Произвольные значения

Функция `slog.Any` принимает ключ и значение любого типа, а затем создаёт из них `slog.Attr`:

```go
type JobResult struct {
    JobID    string
    Attempts int
    Success  bool
}

result := JobResult{
    JobID:    "job-42",
    Attempts: 2,
    Success:  true,
}

logger.Info("job completed",
    slog.Any("result", result),
)
```

То, как значение будет представлено, зависит от обработчика. `JSONHandler` кодирует произвольные значения по правилам `encoding/json`, поэтому экспортируемые поля `JobResult` окажутся во вложенном JSON-объекте. Для краткости служебное поле `time` в примерах вывода опущено:

```json
{"level":"INFO","msg":"job completed","result":{"JobID":"job-42","Attempts":2,"Success":true}}
```

Это удобно для разового значения, но имена полей структуры становятся частью формата записи. Если схема логов должна отличаться от JSON-представления типа, её лучше задать явно с помощью группы атрибутов или `LogValuer`.

Функция [`slog.AnyValue`](https://pkg.go.dev/log/slog#AnyValue) выполняет ту же подготовку значения, но возвращает `slog.Value` без ключа. Она нужна, когда значение создаётся отдельно от атрибута, например внутри метода `LogValue`:

```go
value := slog.AnyValue(result)
attr := slog.Attr{
    Key:   "result",
    Value: value,
}
```

Для встроенных типов предпочтительнее специализированные функции вроде `slog.String` и `slog.Int`: они сразу показывают ожидаемый тип поля. `slog.Any` остаётся общим вариантом для значений, для которых отдельного конструктора нет.

## Ошибка как обычное значение

Для типа `error` в `slog` нет отдельного атрибута. Ошибка передаётся через `slog.Any`, как любое другое значение:

```go
logger.Error("job failed",
    slog.String("job_id", jobID),
    slog.Any("err", err),
)
```

Встроенные обработчики представляют ошибку верхнего уровня через её метод `Error`. Вызов `logger.Error` задаёт уровень события, но не добавляет стек вызовов и не раскрывает цепочку ошибок отдельными полями.

Предварительно вызывать `err.Error()` обычно не нужно. Исходное значение сохраняется в атрибуте до обработки записи, поэтому собственный обработчик или тип ошибки, реализующий `LogValuer`, может определить другое представление.

## Интерфейс `LogValuer`

Интерфейс [`slog.LogValuer`](https://pkg.go.dev/log/slog#LogValuer) состоит из одного метода:

```go
type LogValuer interface {
    LogValue() slog.Value
}
```

Когда значение реализует этот интерфейс, обработчик использует результат `LogValue` вместо обычного форматирования структуры. Например, прикладной тип может задать стабильные имена полей независимо от имён и JSON-тегов исходной структуры:

```go
type JobResult struct {
    JobID    string
    Attempts int
    Success  bool
}

func (r JobResult) LogValue() slog.Value {
    return slog.GroupValue(
        slog.String("job_id", r.JobID),
        slog.Int("attempts", r.Attempts),
        slog.Bool("success", r.Success),
    )
}
```

После этого обычный вызов `slog.Any` использует новое представление:

```go
logger.Info("job completed",
    slog.Any("result", result),
)
```

`JSONHandler` сформирует вложенную группу с ключами, заданными в `LogValue`:

```json
{"level":"INFO","msg":"job completed","result":{"job_id":"job-42","attempts":2,"success":true}}
```

Код, создающий событие, не перечисляет поля результата, но схема остаётся явной: она находится рядом с определением прикладного типа.

## Структурное значение через `GroupValue`

Функция [`slog.GroupValue`](https://pkg.go.dev/log/slog#GroupValue) создаёт `slog.Value`, содержащий несколько атрибутов. В отличие от `slog.Group`, она не принимает имя группы: внешним ключом становится ключ атрибута, через который значение передано логгеру.

```go
func (r JobResult) LogValue() slog.Value {
    return slog.GroupValue(
        slog.String("job_id", r.JobID),
        slog.Int("attempts", r.Attempts),
        slog.Bool("success", r.Success),
    )
}

logger.Info("job completed",
    slog.Any("result", result),
)
```

В этом примере `result` задаёт имя группы, а `job_id`, `attempts` и `success` становятся её полями. Если `LogValue` должен вернуть одно значение, используется соответствующий конструктор `slog.Value`:

```go
type Region string

func (r Region) LogValue() slog.Value {
    return slog.StringValue(string(r))
}
```

Выбор между одиночным значением и `GroupValue` зависит от смысла типа. Идентификатор или перечисление обычно достаточно представить строкой, а объект из нескольких связанных полей — группой.

## Безопасное представление типа

`LogValuer` особенно полезен для типов, которые содержат как диагностические, так и чувствительные данные. Вместо передачи всей структуры можно вернуть только заранее разрешённые поля:

```go
type UpstreamCredential struct {
    Service   string
    Token     string
    ExpiresAt time.Time
}

func (c UpstreamCredential) LogValue() slog.Value {
    return slog.GroupValue(
        slog.String("service", c.Service),
        slog.Time("expires_at", c.ExpiresAt),
    )
}
```

Значение по-прежнему передаётся обычным способом:

```go
logger.Info("upstream credential loaded",
    slog.Any("credential", credential),
)
```

Результат содержит сведения, полезные для диагностики, но не содержит `Token`:

```json
{"level":"INFO","msg":"upstream credential loaded","credential":{"service":"payments","expires_at":"2026-07-22T12:00:00Z"}}
```

::: warning
`LogValuer` управляет только представлением значения целиком. Если код отдельно передаст `credential.Token` в другой атрибут, интерфейс не сможет его скрыть. Безопасность по-прежнему зависит от явного выбора данных в местах создания событий.
:::

## Момент разрешения значения

Вызов `slog.Any` не вызывает `LogValue` немедленно. Значение разрешается обработчиком после проверки уровня, поэтому для отключённого события вычисление обычно не выполняется. Это позволяет отложить дорогую подготовку данных:

```go
type DeferredSummary struct {
    Build func() string
}

func (s DeferredSummary) LogValue() slog.Value {
    return slog.StringValue(s.Build())
}

logger.Debug("cache state",
    slog.Any("summary", DeferredSummary{
        Build: buildCacheSummary,
    }),
)
```

Если уровень `Debug` отключён, встроенный обработчик не вызовет `buildCacheSummary`. При этом аргументы функции логгера вычисляются по обычным правилам Go: создание `DeferredSummary` произойдёт до вызова `logger.Debug`, а отложена будет только работа внутри `LogValue`.

Атрибуты, добавленные через `Logger.With`, передаются обработчику при создании производного логгера. Обработчик вправе разрешить и подготовить их сразу, поэтому `LogValuer` не следует использовать в `With` для значения, которое должно вычисляться заново при каждом событии.

## Ограничения `LogValuer`

Метод `LogValue` лучше делать детерминированным и без побочных эффектов. Логгер может использоваться конкурентно, а обработчик не обязан вызывать метод ровно один раз, поэтому результат не должен зависеть от счётчика вызовов или изменять состояние приложения.

Метод не должен возвращать то же значение, которое снова реализует `LogValuer`:

```go
func (r JobResult) LogValue() slog.Value {
    // Ошибка: результат снова содержит JobResult с тем же методом LogValue.
    return slog.AnyValue(r)
}
```

[`slog.Value.Resolve`](https://pkg.go.dev/log/slog#Value.Resolve) повторно разрешает вложенную цепочку `LogValuer`, но ограничивает число вызовов, чтобы такая рекурсия не стала бесконечной. Вместо исходного типа нужно вернуть конечное значение: строку, число, время или группу.

Встроенные `TextHandler` и `JSONHandler` вызывают `Resolve` автоматически. Автор собственного обработчика должен сделать это явно перед чтением значения:

```go
value := attr.Value.Resolve()
```

Паника внутри `LogValue` преобразуется `Resolve` в диагностическое значение, но полагаться на это как на обработку ошибок не стоит. `LogValue` не может вернуть `error`, поэтому потенциально ошибочное или блокирующее вычисление лучше выполнить заранее, а в лог передать уже готовый результат.
