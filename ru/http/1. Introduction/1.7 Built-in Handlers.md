# Встроенные обработчики

Стандартная библиотека [`net/http`](https://pkg.go.dev/net/http) предоставляет готовые компоненты, совместимые с интерфейсом [`http.Handler`](https://pkg.go.dev/net/http#Handler). Они важны не только как утилиты, но и как пример общей архитектуры пакета: разные задачи можно подключать через один и тот же контракт `ServeHTTP`.

Это не официальная классификация стандартной библиотеки, но для обзора встроенные обработчики удобно сгруппировать по роли:

- **обработчики-обёртки** — принимают один `http.Handler` и возвращают другой `http.Handler`, добавляя поведение вокруг него;
- **готовые ответы** — сразу формируют ответ и обычно становятся последним звеном цепочки.

## Обработчики-обёртки

Обработчики-обёртки расширяют уже существующий обработчик. Этот стиль совпадает с общей моделью middleware: внешний обработчик получает запрос, выполняет свою инфраструктурную работу и передает управление вложенному обработчику.

```go
mux := http.NewServeMux()
mux.HandleFunc("POST /upload", uploadHandler)

// Ограничение чтения тела запроса для всех маршрутов внутри mux.
handler := http.MaxBytesHandler(mux, 2<<20)
```

К типичным компонентам-обёрткам относятся:

| Компонент                                                             | Назначение                                                                                       |
| :-------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------- |
| [`http.MaxBytesHandler`](https://pkg.go.dev/net/http#MaxBytesHandler) | Ограничивает количество байт, которые можно прочитать из тела запроса.                           |
| [`http.TimeoutHandler`](https://pkg.go.dev/net/http#TimeoutHandler)   | Возвращает `503 Service Unavailable`, если вложенный обработчик не завершился за заданное время. |
| [`http.StripPrefix`](https://pkg.go.dev/net/http#StripPrefix)         | Удаляет префикс пути перед передачей запроса следующему обработчику.                             |

## Раздача файлов

[`http.FileServer`](https://pkg.go.dev/net/http#FileServer) и [`http.FileServerFS`](https://pkg.go.dev/net/http#FileServerFS) отдают файлы как обычные HTTP-ответы. Чаще всего их комбинируют с `http.StripPrefix`, если URL-префикс не совпадает с путем внутри файловой системы.

```go
mux := http.NewServeMux()

files := http.FileServer(http.Dir("./public"))
mux.Handle("/static/", http.StripPrefix("/static/", files))
```

`http.FileServerFS` решает ту же задачу для файловых систем на базе [`io/fs.FS`](https://pkg.go.dev/io/fs#FS), например для файлов, встроенных через пакет [`embed`](https://pkg.go.dev/embed).

::: warning
Через `FileServer` не следует раздавать корень проекта, директории с конфигурацией, секретами или внутренними файлами. Для статических ресурсов лучше выделять отдельную папку вроде `public` или `assets`.
:::

## Готовые ответы

Некоторые встроенные компоненты сразу формируют ответ и не требуют вложенного обработчика.

### http.RedirectHandler

[`http.RedirectHandler`](https://pkg.go.dev/net/http#RedirectHandler) возвращает обработчик, который отвечает редиректом на каждый входящий запрос.

```go
mux := http.NewServeMux()
mux.Handle("/old", http.RedirectHandler("/new", http.StatusMovedPermanently))
```

Для временных редиректов обычно используют `302` или `307`, для постоянных — `301` или `308`. Коды `307` и `308` сохраняют HTTP-метод, поэтому они безопаснее для API-сценариев, где нельзя превращать `POST` в `GET`.

### http.NotFoundHandler

[`http.NotFoundHandler`](https://pkg.go.dev/net/http#NotFoundHandler) возвращает обработчик, который всегда отвечает `404 Not Found`.

```go
mux := http.NewServeMux()
mux.Handle("/internal/", http.NotFoundHandler())
```

`ServeMux` и сам вернет `404` для неизвестных маршрутов, но `NotFoundHandler` полезен, когда нужно явно закрыть ветку URL или встроить стандартный ответ в более сложную маршрутизацию. Подробная настройка маршрутов разбирается в статье [Маршрутизация](/ru/http/server/routing).
