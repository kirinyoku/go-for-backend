# Тип http.HandlerFunc

Реализация [`http.Handler`](https://pkg.go.dev/net/http#Handler) через структуру требует отдельного типа с методом [`ServeHTTP`](https://pkg.go.dev/net/http#Handler.ServeHTTP). Для простых обработчиков, которым не нужно хранить внутреннее состояние, стандартная библиотека Go предоставляет тип [`http.HandlerFunc`](https://pkg.go.dev/net/http#HandlerFunc). Он позволяет использовать обычные функции в качестве обработчиков запросов и сокращает количество шаблонного кода.

## Определение HandlerFunc

`http.HandlerFunc` — это именованный тип на базе функциональной сигнатуры. Он реализует интерфейс `http.Handler` через метод `ServeHTTP`, который просто вызывает саму функцию.

```go
type HandlerFunc func(http.ResponseWriter, *http.Request)

// ServeHTTP вызывает f(w, r), позволяя типу соответствовать интерфейсу Handler.
func (f HandlerFunc) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    f(w, r)
}
```

## Преобразование функции к http.Handler

Использование `http.HandlerFunc` представляет собой операцию преобразования типа (*type conversion*), а не вызов функции.

```go
func healthCheck(w http.ResponseWriter, r *http.Request) {
    w.WriteHeader(http.StatusOK)
    fmt.Fprint(w, "OK")
}

// http.HandlerFunc адаптирует обычную функцию к интерфейсу http.Handler.
var handler http.Handler = http.HandlerFunc(healthCheck)
```

Сама функция `healthCheck` не реализует `http.Handler`, но после преобразования к `http.HandlerFunc` значение получает метод `ServeHTTP` и становится совместимым с интерфейсом.

## Внедрение зависимостей

Замыкания позволяют передавать внешние значения в обработчик без глобальных переменных. Функция-конструктор принимает параметры конфигурации и возвращает `http.HandlerFunc`.

```go
func NewHealthHandler(version string) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Content-Type", "text/plain")

        // Обработка ошибки записи ответа опущена.
        fmt.Fprintf(w, "OK %s", version)
    }
}
```

::: tip
Замыкания удобны для небольших обработчиков и middleware, где зависимостей мало и поведение помещается в одну функцию. Если обработчик растет, использует несколько зависимостей или требует набора связанных методов, удобнее перейти к структуре с методом `ServeHTTP`.
:::

## HandlerFunc в middleware

Middleware часто описывается как функция, которая принимает `http.Handler` и возвращает новый `http.Handler`. `http.HandlerFunc` удобен для таких оберток: он позволяет вернуть обычную функцию там, где ожидается интерфейс.

```go
func LoggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        log.Printf("%s %s", r.Method, r.URL.Path)
        next.ServeHTTP(w, r)
    })
}
```

::: info
Подробный разбор middleware находится в статье [Middleware](/ru/net-http/server/middleware).
:::
