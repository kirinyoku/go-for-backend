# Интерфейс http.ResponseWriter

Интерфейс [`http.ResponseWriter`](https://pkg.go.dev/net/http#ResponseWriter) является основным способом записать HTTP-ответ в серверном обработчике. Через него handler задаёт заголовки, выбирает статус-код и отправляет тело ответа.

`ResponseWriter` не хранит готовый ответ как структура данных. Это интерфейс записи, действующий только во время выполнения обработчика. Готовый ответ представлен другой структурой — [`http.Response`](/ru/net-http/intro/response), которая чаще встречается в клиентском коде и тестах.

## Основные методы

В отличие от [`http.Request`](https://pkg.go.dev/net/http#Request), который является структурой, `ResponseWriter` — это интерфейс. Это позволяет пакету [`net/http`](https://pkg.go.dev/net/http) использовать различные внутренние реализации в зависимости от версии протокола HTTP и конфигурации сервера.

```go
type ResponseWriter interface {
    Header() Header
    WriteHeader(statusCode int)
    Write([]byte) (int, error)
}
```

| Метод                                                                        | Описание                                                 |
| :--------------------------------------------------------------------------- | :------------------------------------------------------- |
| [`Header() Header`](https://pkg.go.dev/net/http#ResponseWriter.Header)       | Возвращает карту заголовков для последующей модификации. |
| [`WriteHeader(int)`](https://pkg.go.dev/net/http#ResponseWriter.WriteHeader) | Отправляет клиенту код состояния HTTP.                   |
| [`Write([]byte)`](https://pkg.go.dev/net/http#ResponseWriter.Write)          | Записывает данные в тело ответа и фиксирует заголовки.   |

## Порядок выполнения операций

Протокол HTTP требует строгого порядка передачи данных: сначала передаются заголовки и статус-код, затем — тело ответа. В стандартной библиотеке Go этот порядок соблюдается автоматически, но это накладывает на разработчика важные ограничения.

```mermaid
flowchart LR
    A[Header.Set] --> B[WriteHeader]
    B --> C[Write]
    C --> D[Завершение]
```

### 1. Модификация заголовков

Метод `Header()` возвращает [`http.Header`](https://pkg.go.dev/net/http#Header) — карту заголовков, которые будут отправлены клиенту. Все изменения должны быть завершены до первого вызова `Write` или `WriteHeader`.

```go
func handler(w http.ResponseWriter, r *http.Request) {
    // Установка заголовка должна быть первым действием.
    w.Header().Set("Content-Type", "application/json")
    
    // ... логика
}
```

### 2. Установка статуса

Метод `WriteHeader` фиксирует код состояния ответа.

- Если метод не вызван явно, при первой записи в тело будет отправлен статус `200 OK`.
- После вызова `WriteHeader` обычные HTTP-заголовки считаются отправленными клиенту, поэтому их дальнейшее изменение не влияет на ответ.

### 3. Запись тела

Метод `Write` отправляет данные клиенту.

- Если статус ещё не был отправлен, первый вызов `Write` автоматически вызывает `WriteHeader(http.StatusOK)`.
- Если заголовок `Content-Type` не задан вручную, библиотека выполняет анализ первых 512 байт данных для автоматического определения типа контента (*content sniffing*).

```go
func handler(w http.ResponseWriter, r *http.Request) {
    // 1. Заголовки
    w.Header().Set("Content-Type", "text/plain")
    // 2. Статус код
    w.WriteHeader(http.StatusAccepted)
    
    // 3. Запись в тело ответа.
    if _, err := w.Write([]byte("Request accepted")); err != nil {
        log.Printf("failed to write response: %v", err)
    }
}
```

::: warning
Любая попытка изменить заголовки после вызова `Write` является логической ошибкой: сервер проигнорирует эти изменения, так как HTTP-заголовки уже отправлены в сетевой стек. Повторный вызов `WriteHeader` также не изменит статус ответа, а в логах сервера появится предупреждение `http: superfluous response.WriteHeader call`.
:::

## Частые ошибки

Вызов [`http.Error`](https://pkg.go.dev/net/http#Error), `WriteHeader` или `Write` фиксирует ответ, но не останавливает выполнение функции. Если после этого текущая ветка обработки завершена, нужно явно выйти из обработчика через `return`. Иначе код ниже продолжит выполняться и может попытаться записать другой статус или тело.

```go
func handler(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "method not allowed", http.StatusMethodNotAllowed)
        return
    }

    fmt.Fprint(w, "created")
}
```

Также важно не сохранять `ResponseWriter` в глобальные переменные и не использовать его после возврата из обработчика. Он привязан к конкретному запросу и жизненному циклу соединения.

## Расширенные возможности

Интерфейс `http.ResponseWriter` намеренно минималистичен. Для потоковой передачи, управления дедлайнами и низкоуровневых операций в Go 1.20+ используется [`http.ResponseController`](https://pkg.go.dev/net/http#ResponseController). Он позволяет вызвать операции вроде `Flush`, `SetReadDeadline`, `SetWriteDeadline` и `Hijack`, если текущая реализация ответа их поддерживает.

Для обычных JSON/API-обработчиков достаточно базовых методов `Header`, `WriteHeader` и `Write`. Расширенные операции становятся важны позже: когда нужно сбрасывать буфер потокового ответа, управлять deadline чтения и записи, забирать соединение через `Hijack` или сохранить эти возможности после обёртки `ResponseWriter` в middleware. Эти сценарии разбираются дальше, в статье [Потоковые ответы и http.ResponseController](/ru/net-http/server/responsecontroller).
