# Маршрутизация

[`http.ServeMux`](https://pkg.go.dev/net/http#ServeMux) — стандартный мультиплексор (маршрутизатор) пакета [`net/http`](https://pkg.go.dev/net/http). Он реализует интерфейс [`http.Handler`](https://pkg.go.dev/net/http#Handler) и отвечает за сопоставление входящих запросов с зарегистрированными обработчиками на основе URL-паттернов.

Начиная с Go 1.22, возможности `http.ServeMux` были расширены: добавлена встроенная поддержка HTTP-методов и именованных сегментов (*wildcards*), что позволяет строить сложные API без использования сторонних библиотек.

## Регистрация обработчиков

Для регистрации маршрутов используются методы [`Handle`](https://pkg.go.dev/net/http#ServeMux.Handle) и [`HandleFunc`](https://pkg.go.dev/net/http#ServeMux.HandleFunc). Разница между ними заключается в типе принимаемого аргумента:

```go
mux := http.NewServeMux()

// Handle принимает объект, реализующий интерфейс http.Handler.
mux.Handle("/api/v1/", apiHandler)

// HandleFunc принимает функцию с сигнатурой обработчика.
mux.HandleFunc("GET /health", func(w http.ResponseWriter, r *http.Request) {
    w.WriteHeader(http.StatusOK)
    fmt.Fprintln(w, "OK")
})
```

::: tip
Для приложений используйте локальный `http.NewServeMux()`, а не глобальный `http.DefaultServeMux`.
:::

## Паттерны маршрутов в Go 1.22+

Паттерн может включать HTTP-метод, путь и специальные символы для захвата параметров.

### 1. HTTP-методы

Метод указывается в начале строки. Если он опущен, маршрут соответствует любому методу. Регистрация `GET` автоматически разрешает запросы `HEAD`.

```go
mux.HandleFunc("GET /articles", listHandler)    // Только GET
mux.HandleFunc("POST /articles", createHandler) // Только POST
mux.HandleFunc("/public/", fileHandler)         // Любой метод
```

### 2. Именованые сегменты (Wildcards)

Именованные сегменты позволяют динамически извлекать части пути.

- `{name}` — соответствует одному сегменту пути.
- `{name...}` — захватывает все оставшиеся сегменты.

```go
mux.HandleFunc("GET /users/{id}", func(w http.ResponseWriter, r *http.Request) {
    // PathValue возвращает значение заполнителя в виде строки.
    idStr := r.PathValue("id")
    
    // Обязательная валидация и преобразование типов.
    id, err := strconv.Atoi(idStr)
    if err != nil {
        http.Error(w, "invalid user id", http.StatusBadRequest)
        return
    }
    
    fmt.Fprintf(w, "User ID: %d\n", id)
})
```

### 3. Точные и префиксные совпадения

| Паттерн | Тип совпадения | Описание |
| :--- | :--- | :--- |
| `/path` | Точное | Соответствует только `/path`. |
| `/path/` | Префиксное | Соответствует `/path/` и любым вложенным путям (напр. `/path/sub`). |
| `/path/{$}` | Точное | Соответствует только `/path/` (со слешем в конце). |

## Приоритет маршрутов

`http.ServeMux` выбирает обработчик на основе правила **наибольшей специфичности**. Порядок регистрации маршрутов в коде не влияет на результат. Паттерн считается более специфичным, если он описывает более узкое подмножество запросов.

| Критерий           | Пример                      | Обоснование                                     |
| :----------------- | :-------------------------- | :---------------------------------------------- |
| **Наличие метода** | `GET /` > `/`               | Указание метода сужает область поиска.          |
| **Тип сегмента**   | `/users/me` > `/users/{id}` | Константа специфичнее заполнителя.              |
| **Длина пути**     | `/a/b/c` > `/a/b/`          | Большее количество сегментов сужает совпадение. |

::: warning
Если пути пересекаются, но ни один из паттернов не является более специфичным (строгим подмножеством другого), мультиплексор вызывает `panic` для предотвращения неопределённости. Классический пример — `GET /` и `/index.html`. Первый перехватывает все GET-запросы, второй перехватывает запросы к `/index.html` с любым методом. При запросе `GET /index.html` возникает неопределённость, так как ни одно из правил не является более специфичным.
:::

## Вложенная маршрутизация

Поскольку мультиплексор сам является `http.Handler`, маршруты можно группировать, вкладывая один `ServeMux` в другой. Это полезно для версионирования API или применения middleware к определенным веткам.

```go
// Создание изолированной группы маршрутов.
api := http.NewServeMux()
api.HandleFunc("GET /users", listUsers)

root := http.NewServeMux()
// StripPrefix удаляет префикс перед передачей запроса во вложенный роутер.
root.Handle("/api/v1/", http.StripPrefix("/api/v1", api))
```

::: warning
Если зарегистрирован путь с завершающим слешем (например, `/api/`), запросы без него (`/api`) будут автоматически перенаправлены (статус `301`). Это может привести к потере тела запроса в `POST/PUT` методах у некоторых клиентов.
:::

## Потокобезопасность

`http.ServeMux` безопасен для конкурентного использования: методы `Handle` и `HandleFunc` защищены мьютексом. Однако рекомендуется регистрировать все маршруты на этапе инициализации до запуска сервера.
