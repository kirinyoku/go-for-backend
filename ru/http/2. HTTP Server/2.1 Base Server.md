# Запуск базового сервера

Пакет [`net/http`](https://pkg.go.dev/net/http) позволяет запустить базовый HTTP-сервер двумя способами: через высокоуровневую функцию [`http.ListenAndServe`](https://pkg.go.dev/net/http#ListenAndServe) для быстрого старта или через структуру [`http.Server`](https://pkg.go.dev/net/http#Server), когда нужен полный контроль над конфигурацией и поведением.

## Быстрый старт: http.ListenAndServe

Для запуска сервера в одну строку используется функция `http.ListenAndServe`. Она автоматически создает `http.Server` с настройками по умолчанию и начинает прослушивание указанного адреса.

```go
package main

import (
    "fmt"
    "log"
    "net/http"
)

func main() {
    // Регистрация обработчика в глобальном маршрутизаторе DefaultServeMux.
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "Hello, World!")
    })

    // ListenAndServe блокирует выполнение.
    // При нормальной работе функция никогда не возвращает nil.
    err := http.ListenAndServe(":8080", nil)
    if err != nil {
        log.Fatalf("server failed: %s", err)
    }
}
```

Вторым аргументом в `http.ListenAndServe` передается реализация интерфейса [`http.Handler`](https://pkg.go.dev/net/http#Handler). Значение `nil` указывает на использование [`http.DefaultServeMux`](https://pkg.go.dev/net/http#DefaultServeMux) — глобального маршрутизатора пакета.

### Ограничения функции

Использование `http.ListenAndServe` и глобального состояния не рекомендуется по следующим причинам:

- **Глобальное состояние:** `http.DefaultServeMux` доступен для модификации любому импортированному пакету. Это может привести к появлению нежелательных эндпоинтов (например, при импорте пакета [`net/http/pprof`](https://pkg.go.dev/net/http/pprof)).
- **Отсутствие таймаутов:** По умолчанию таймауты не установлены, что делает сервер уязвимым к атакам.

## Запуск через http.Server

Структура `http.Server` — основной инструмент для построения надежных сервисов. Она позволяет явно управлять таймаутами, лимитами и параметрами TLS.

```go
package main

import (
    "errors"
    "log"
    "net/http"
    "time"
)

func main() {
    // Cоздание локального маршрутизатора.
    mux := http.NewServeMux()
    mux.HandleFunc("GET /health", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
    })

    server := &http.Server{
        Addr:    ":8080",
        // Явное использование локального маршрутизатора mux
        // вместо глобального http.DefaultServeMux.
        Handler: mux,

        // Максимальное время на чтение заголовков запроса.
        ReadHeaderTimeout: 5 * time.Second,
        // Общее время на чтение всего запроса (заголовки + тело).
        ReadTimeout:       10 * time.Second,
        // Время на выполнение обработчика и отправку ответа (начинается после чтения заголовков).
        WriteTimeout:      10 * time.Second,
        // Время ожидания следующего запроса в Keep-Alive соединении.
        IdleTimeout:       120 * time.Second,
    }

    log.Printf("server listening on %s", server.Addr)

    // Проверка на http.ErrServerClosed позволяет отличить плановую остановку от ошибки.
    if err := server.ListenAndServe(); err != nil && !errors.Is(err, http.ErrServerClosed) {
        log.Fatalf("server listen error: %s", err)
    }
}
```

::: info
`http.Server` не содержит поля для ограничения размера тела запроса (например, для защиты от загрузки слишком больших файлов). Это ограничение реализуется на уровне обработчиков с помощью [`http.MaxBytesReader`](https://pkg.go.dev/net/http#MaxBytesReader). Подробнее см. в статье [Ограничение ресурсов](./resource-limits).
:::

### Параметры конфигурации

| Поле                | Назначение                                                                 |
| :------------------ | :------------------------------------------------------------------------- |
| `Addr`              | Адрес прослушивания (`host:port`).                                         |
| `Handler`           | Основной обработчик запросов (обычно маршрутизатор).                       |
| `ReadHeaderTimeout` | Лимит времени на получение заголовков. Предотвращает удержание соединений. |
| `ReadTimeout`       | Лимит на чтение всего запроса.                                             |
| `WriteTimeout`      | Лимит на формирование и отправку ответа.                                   |
| `IdleTimeout`       | Лимит на ожидание нового запроса в открытом соединении.                    |
| `MaxHeaderBytes`    | Максимальный размер заголовков. По умолчанию 1 МБ.                         |
| `TLSConfig`         | Конфигурация для работы по протоколу HTTPS.                                |

## Безопасное соединение (HTTPS)

Для работы по протоколу HTTPS используется метод [`ListenAndServeTLS`](https://pkg.go.dev/net/http#Server.ListenAndServeTLS). Конфигурация сервера остается идентичной, за исключением указания путей к сертификатам. Детальная настройка HTTPS описана в статье [HTTPS](./tls-and-https).

```go
server := &http.Server{
    Addr:              ":443",
    Handler:           mux,
    ReadHeaderTimeout: 5 * time.Second,
}

// cert.pem и key.pem — файлы сертификата и приватного ключа.
err := server.ListenAndServeTLS("cert.pem", "key.pem")
if err != nil && !errors.Is(err, http.ErrServerClosed) {
    log.Fatalf("tls server failed: %s", err)
}
```

При активации TLS пакет `net/http` автоматически настраивает поддержку протокола HTTP/2 через механизм ALPN.
