# Ограничение ресурсов

[`http.Server`](https://pkg.go.dev/net/http#Server) не знает, какие размеры запросов и задержки нормальны для конкретного приложения. Поэтому сервер нужно настраивать явно: ограничивать время чтения запроса, время записи ответа, время простоя keep-alive соединений, размер заголовков и размер тела запроса.

Без этих лимитов медленный или слишком большой запрос может долго удерживать соединение, горутину, память и файловый дескриптор.

## Лимиты http.Server

Основные лимиты задаются полями структуры `http.Server`:

| Поле | Что ограничивает |
| :-- | :-- |
| [`ReadTimeout`](https://pkg.go.dev/net/http#Server.ReadTimeout) | Максимальное время чтения всего запроса, включая заголовки и тело |
| [`WriteTimeout`](https://pkg.go.dev/net/http#Server.WriteTimeout) | Максимальное время записи ответа |
| [`IdleTimeout`](https://pkg.go.dev/net/http#Server.IdleTimeout) | Максимальное время ожидания следующего запроса на keep-alive соединении |
| [`MaxHeaderBytes`](https://pkg.go.dev/net/http#Server.MaxHeaderBytes) | Максимальный размер заголовков запроса |

```go
server := &http.Server{
    Addr: ":8080",

    // ReadTimeout ограничивает чтение всего запроса:
    // start-line, headers и body.
    ReadTimeout: 10 * time.Second,

    // WriteTimeout ограничивает запись ответа клиенту.
    // Для streaming-эндпоинтов значение нужно выбирать осторожно.
    WriteTimeout: 15 * time.Second,

    // IdleTimeout ограничивает простой keep-alive соединения
    // между завершённым ответом и следующим запросом.
    IdleTimeout: 120 * time.Second,

    // MaxHeaderBytes ограничивает суммарный размер заголовков.
    // Если поле не задано, используется http.DefaultMaxHeaderBytes: 1 МБ.
    MaxHeaderBytes: 1 << 20,
}
```

::: tip
Таймауты защищают не только от злоумышленников, но и от обычных сетевых сбоев: медленного клиента, зависшего соединения или клиента, который начал отправлять запрос и перестал передавать данные.
:::

## ReadTimeout

`ReadTimeout` задаёт общий deadline на чтение входящего запроса. В этот интервал входит чтение заголовков и тела.

Это поле полезно, когда клиент отправляет данные слишком медленно: например, долго передаёт большое тело формы или не завершает отправку JSON. Если deadline истекает, сервер прекращает чтение запроса.

```go
server := &http.Server{
    Addr:        ":8080",
    ReadTimeout: 10 * time.Second,
}
```

`ReadTimeout` не ограничивает размер тела. Быстрый клиент всё ещё может отправить слишком большой payload, поэтому для body нужен отдельный лимит через [`http.MaxBytesReader`](https://pkg.go.dev/net/http#MaxBytesReader).

## WriteTimeout

`WriteTimeout` задаёт deadline на запись ответа. Он защищает сервер от ситуации, когда клиент читает ответ слишком медленно и обработчик долго удерживает соединение.

```go
server := &http.Server{
    Addr:         ":8080",
    WriteTimeout: 15 * time.Second,
}
```

Для обычных JSON API это безопасный лимит. Для потоковых ответов, например Server-Sent Events или долгой передачи файла, слишком маленький `WriteTimeout` может оборвать легитимный ответ. В таких обработчиках лимит нужно выбирать отдельно или не задавать глобально слишком жёстким. Более точечное управление записью ответа разбирается в статье [Потоковые ответы и http.ResponseController](/ru/http/server/responsecontroller).

## IdleTimeout

`IdleTimeout` ограничивает время простоя keep-alive соединения после завершения ответа. Если клиент не отправляет следующий запрос в течение этого времени, сервер закрывает соединение.

```go
server := &http.Server{
    Addr:        ":8080",
    IdleTimeout: 120 * time.Second,
}
```

Если `IdleTimeout` не задан, Go использует значение `ReadTimeout`. Если оба поля равны нулю, idle-соединения могут оставаться открытыми без серверного deadline.

## MaxHeaderBytes

`MaxHeaderBytes` ограничивает суммарный размер заголовков запроса. Если поле не задано, используется [`http.DefaultMaxHeaderBytes`](https://pkg.go.dev/net/http#DefaultMaxHeaderBytes) — 1 МБ.

```go
server := &http.Server{
    Addr:           ":8080",
    MaxHeaderBytes: 1 << 20, // 1 МБ
}
```

Большинству API достаточно значения по умолчанию. Его уменьшают, если нужно жёстче ограничить память на соединение, или увеличивают, если приложение легитимно принимает крупные заголовки, например большие cookies.

## Ограничение тела запроса

У `http.Server` нет поля, которое ограничивает размер body для всех запросов. Размер тела ограничивается на уровне обработчика через `http.MaxBytesReader`.

`MaxBytesReader` оборачивает `r.Body`. Все последующие читатели — `json.Decoder`, `io.ReadAll`, `r.ParseMultipartForm` — будут читать уже из ограниченного потока. При превышении лимита чтение вернёт ошибку типа [`*http.MaxBytesError`](https://pkg.go.dev/net/http#MaxBytesError).

```go
func createPostHandler(w http.ResponseWriter, r *http.Request) {
    const maxBodySize = 1 << 20 // 1 МБ

    r.Body = http.MaxBytesReader(w, r.Body, maxBodySize)
    defer r.Body.Close()

    body, err := io.ReadAll(r.Body)
    if err != nil {
        var maxBytesErr *http.MaxBytesError
        if errors.As(err, &maxBytesErr) {
            http.Error(w, "request body too large", http.StatusRequestEntityTooLarge)
            return
        }

        http.Error(w, "failed to read request body", http.StatusBadRequest)
        return
    }

    // ... обработка body
    _ = body
}
```

Для JSON-обработчиков `MaxBytesReader` ставится перед созданием `json.Decoder`. Для multipart-загрузок — перед вызовом `r.ParseMultipartForm` или `r.FormFile`.
