# Чтение и запись данных

Веб-сервер постоянно читает данные из запроса и записывает данные в ответ: принимает параметры поиска, декодирует JSON, обрабатывает HTML-формы, получает файлы и возвращает результат клиенту. В Go такая работа строится на интерфейсах [`io.Reader`](https://pkg.go.dev/io#Reader) и [`io.Writer`](https://pkg.go.dev/io#Writer) — потоковая модель позволяет обрабатывать тело запроса без его полной загрузки в память. Выбор подходящего механизма для каждого сценария влияет и на производительность, и на безопасность сервера.

## URL-параметры

Параметры строки запроса доступны через [`r.URL.Query()`](https://pkg.go.dev/net/url#URL.Query), возвращающий [`url.Values`](https://pkg.go.dev/net/url#Values) — псевдоним типа `map[string][]string`.

| Метод                                                       | Поведение                                             |
| :---------------------------------------------------------- | :---------------------------------------------------- |
| [`values.Get("key")`](https://pkg.go.dev/net/url#URL.Query) | Возвращает первое значение по ключу или пустую строку |
| `values["key"]`                                             | Возвращает срез всех значений по ключу                |

```go
func searchHandler(w http.ResponseWriter, r *http.Request) {
    query := r.URL.Query()

    // Get возвращает первое значение по ключу или пустую строку.
    q := query.Get("q")

    // Срез всех значений одного ключа: ?tag=go&tag=web → ["go", "web"].
    tags := query["tag"]

    // Все значения query — строки. 
    // Числовые параметры требуют явного преобразования.
    page, err := strconv.Atoi(query.Get("page"))
    if err != nil || page < 1 {
        // Значение по умолчанию.
        page = 1
    }

    fmt.Fprintf(w, "query=%s tags=%v page=%d", q, tags, page)
}
```

::: warning
`strconv.Atoi("")` возвращает `0, err`. Без явной проверки ошибки логика пагинации и фильтрации нарушается незаметно.
:::

## Работа с JSON

JSON — основной формат обмена данными в современных API. [`json.Decoder`](https://pkg.go.dev/encoding/json#Decoder) и [`json.Encoder`](https://pkg.go.dev/encoding/json#Encoder) работают напрямую с `io.Reader`/`io.Writer`, не загружая тело запроса или ответа в память целиком — в отличие от [`json.Unmarshal`](https://pkg.go.dev/encoding/json#Unmarshal)/[`json.Marshal`](https://pkg.go.dev/encoding/json#Marshal).

```go
// json.Decoder читает данные потоково, не буферизуя тело целиком.
if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
    http.Error(w, "Invalid JSON", http.StatusBadRequest)
    return
}
```

### Декодирование запроса

```go
type CreateUserRequest struct {
    Name  string `json:"name"`
    Email string `json:"email"`
}

func createUserHandler(w http.ResponseWriter, r *http.Request) {
    mediaType, _, err := mime.ParseMediaType(r.Header.Get("Content-Type"))
    if err != nil || mediaType != "application/json" {
        http.Error(w, "Content-Type must be application/json", http.StatusUnsupportedMediaType)
        return
    }

    dec := json.NewDecoder(r.Body)
    // DisallowUnknownFields отклоняет JSON с полями, не объявленными в структуре.
    // Защищает от опечаток в именах полей и случайной передачи лишних данных.
    dec.DisallowUnknownFields()

    var req CreateUserRequest
    if err := dec.Decode(&req); err != nil {
        http.Error(w, "Invalid JSON: "+err.Error(), http.StatusBadRequest)
        return
    }

    // json.Decoder может читать несколько JSON-значений из одного потока.  
    // Первый Decode прочитал только первый объект, поэтому вторым Decode  
    // проверяем, что в теле больше нет второго JSON-документа.
    if err := dec.Decode(&struct{}{}); err != io.EOF {
        http.Error(w, "Request body must contain a single JSON object", http.StatusBadRequest)
        return
    }
    
    // ...
}
```

### Вспомогательная функция writeJSON

Установка заголовка `Content-Type`, запись статус-кода и сериализация тела повторяются в каждом JSON-обработчике. Вынесение этих операций в единую вспомогательную функцию устраняет дублирование:

```go
// writeJSON сериализует v в JSON и отправляет ответ с заданным статус-кодом.
// После вызова w.WriteHeader изменить статус-код уже невозможно —
// ошибки кодирования остаётся только залогировать.
func writeJSON(w http.ResponseWriter, status int, v any) {
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    if err := json.NewEncoder(w).Encode(v); err != nil {
        log.Printf("writeJSON: %v", err)
    }
}
```

::: info
[`json.Encoder.Encode`](https://pkg.go.dev/encoding/json#Encoder.Encode) добавляет `\n` в конце вывода. В контексте HTTP это безвредно: JSON-клиенты игнорируют завершающие пробельные символы.
:::

### Формат ошибок JSON API

В примерах часто используется [`http.Error`](https://pkg.go.dev/net/http#Error), потому что он компактно показывает отправку ошибки и досрочный выход из обработчика. Но `http.Error` по умолчанию возвращает ответ с типом `text/plain`, поэтому в реальном JSON API обычно используют единый JSON-формат ошибок.

```go
type errorResponse struct {
    Error string `json:"error"`
}

func writeJSONError(w http.ResponseWriter, status int, message string) {
    writeJSON(w, status, errorResponse{Error: message})
}
```

Такой helper позволяет всем обработчикам возвращать предсказуемое тело ответа:

```json
{"error":"invalid JSON"}
```

В небольших учебных фрагментах `http.Error` остаётся допустимым упрощением. В API, где клиенты ожидают JSON, лучше использовать один формат ошибок и документировать его как часть контракта.

## HTML-формы и urlencoded-данные

Для HTML-форм с `Content-Type: application/x-www-form-urlencoded` данные формы разбираются через [`r.ParseForm()`](https://pkg.go.dev/net/http#Request.ParseForm). Явный вызов нужен, чтобы обработать ошибку разбора: методы [`r.FormValue`](https://pkg.go.dev/net/http#Request.FormValue) и [`r.PostFormValue`](https://pkg.go.dev/net/http#Request.PostFormValue) вызывают `ParseForm` внутренне, но молча игнорируют возникшую ошибку.

Выбор метода доступа зависит от того, нужно ли учитывать query-параметры URL:

| Метод                                                                       | Источник данных                      |
| :-------------------------------------------------------------------------- | :----------------------------------- |
| [`r.FormValue(key)`](https://pkg.go.dev/net/http#Request.FormValue)         | Тело формы **и** query-параметры URL |
| [`r.PostFormValue(key)`](https://pkg.go.dev/net/http#Request.PostFormValue) | Только тело запроса                  |

```go
func formHandler(w http.ResponseWriter, r *http.Request) {
    if err := r.ParseForm(); err != nil {
        http.Error(w, "Parse error", http.StatusBadRequest)
        return
    }

    // FormValue ищет значение в теле формы и в query-параметрах URL.
    name := r.FormValue("name")

    // PostFormValue ищет значение только в теле запроса, игнорируя query-параметры.
    // Предпочтительнее там, где нужна строгая изоляция источника данных.
    email := r.PostFormValue("email")

    fmt.Fprintf(w, "name=%s email=%s", name, email)
}
```

## Загрузка файлов

Загрузка файлов реализуется через протокол `multipart/form-data`. При обработке загрузок необходимо ограничивать размер входящих данных и проверять реальное содержимое файла по его байтовой сигнатуре — расширение, заявленное клиентом, ненадёжно.

Структура [`multipart.FileHeader`](https://pkg.go.dev/mime/multipart#FileHeader) содержит метаданные о загруженном файле:

| Поле       | Тип                                                                   | Описание                       |
| :--------- | :-------------------------------------------------------------------- | :----------------------------- |
| `Filename` | `string`                                                              | Имя файла, заявленное клиентом |
| `Size`     | `int64`                                                               | Размер файла в байтах          |
| `Header`   | [`textproto.MIMEHeader`](https://pkg.go.dev/net/textproto#MIMEHeader) | Заголовки MIME-части           |

::: warning
`Filename` не следует использовать как имя файла на диске: оно может содержать разделители пути и специальные символы ([атака path traversal](https://en.wikipedia.org/wiki/Directory_traversal_attack)). На диске файл сохраняется под уникальным именем, сгенерированным на сервере.
:::

```go
// uuid "github.com/google/uuid"

func uploadHandler(w http.ResponseWriter, r *http.Request) {
    // Ограничение размера тела предотвращает исчерпание памяти
    // при загрузке слишком больших файлов.
    r.Body = http.MaxBytesReader(w, r.Body, 50<<20) // 50 МБ

    // FormFile возвращает первый файл для данного ключа.
    file, header, err := r.FormFile("upload")
    if err != nil {
        http.Error(w, "Invalid upload", http.StatusBadRequest)
        return
    }
    defer file.Close()

    // Определение MIME-типа по байтовой сигнатуре файла (content sniffing).
    buf := make([]byte, 512)
    n, err := file.Read(buf)
    if err != nil && err != io.EOF {
        http.Error(w, "Read error", http.StatusInternalServerError)
        return
    }

    mimeType := http.DetectContentType(buf[:n])
    switch mimeType {
    case "image/jpeg", "image/png":
        // допустимые типы
    default:
        http.Error(w, "Unsupported file type", http.StatusUnsupportedMediaType)
        return
    }

    // После чтения для проверки типа файловый указатель смещён на 512 байт.
    // Сброс в начало необходим перед копированием файла целиком.
    if _, err := file.Seek(0, io.SeekStart); err != nil {
        http.Error(w, "Seek error", http.StatusInternalServerError)
        return
    }

    // header.Filename логируется для аудита, но не используется как имя файла на диске.
    // UUID исключает перезапись существующих файлов и атаки типа path traversal.
    log.Printf("uploading file: %s (%d bytes)", header.Filename, header.Size)

    dst, err := os.Create(filepath.Join("uploads", uuid.NewString()))
    if err != nil {
        http.Error(w, "Server error", http.StatusInternalServerError)
        return
    }
    defer dst.Close()

    if _, err := io.Copy(dst, file); err != nil {
        http.Error(w, "Save error", http.StatusInternalServerError)
        return
    }
}
```

::: warning
[`r.FormFile`](https://pkg.go.dev/net/http#Request.FormFile) вызывает [`r.ParseMultipartForm`](https://pkg.go.dev/net/http#Request.ParseMultipartForm) с лимитом 32 МБ для хранения данных в оперативной памяти. Данные сверх лимита сохраняются во временных файлах на диске. Для явного управления лимитом или одновременного доступа ко всем загруженным файлам через `r.MultipartForm` следует вызвать `r.ParseMultipartForm(maxMemory)` до обращения к `r.FormFile`.
:::

## Ограничение размера тела запроса

По умолчанию [`net/http`](https://pkg.go.dev/net/http) не ограничивает размер тела запроса. Один запрос с многогигабайтным телом способен исчерпать оперативную память сервера.

[`http.MaxBytesReader`](https://pkg.go.dev/net/http#MaxBytesReader) оборачивает [`r.Body`](https://pkg.go.dev/net/http#Request.ParseMultipartForm) и прекращает чтение при превышении лимита. Возвращаемая ошибка имеет тип [`*http.MaxBytesError`](https://pkg.go.dev/net/http#MaxBytesError), что позволяет отличить её от других сбоев и вернуть клиенту корректный статус.

```go
func restrictedHandler(w http.ResponseWriter, r *http.Request) {
    const maxBytes = 1 << 20 // 1 МБ

    // MaxBytesReader заменяет r.Body: последующие читатели (json.Decoder, io.ReadAll)
    // автоматически получат ошибку при превышении лимита.
    r.Body = http.MaxBytesReader(w, r.Body, maxBytes)

    var data map[string]any
    if err := json.NewDecoder(r.Body).Decode(&data); err != nil {
        // errors.As позволяет различить превышение лимита и ошибку разбора JSON,
        // чтобы вернуть клиенту семантически корректный статус-код.
        var maxBytesErr *http.MaxBytesError
        if errors.As(err, &maxBytesErr) {
            http.Error(w, "Request body too large", http.StatusRequestEntityTooLarge)
            return
        }
        http.Error(w, "Invalid JSON", http.StatusBadRequest)
        return
    }

    // ...
}
```
