# HTTPS

Пакет `net/http` умеет запускать HTTPS-сервер без внешнего веб-сервера. Для этого используется тот же [`http.Server`](https://pkg.go.dev/net/http#Server), но вместо [`ListenAndServe`](https://pkg.go.dev/net/http#Server.ListenAndServe) вызывается [`ListenAndServeTLS`](https://pkg.go.dev/net/http#Server.ListenAndServeTLS).

HTTPS здесь означает, что HTTP работает поверх TLS. За параметры TLS отвечает пакет [`crypto/tls`](https://pkg.go.dev/crypto/tls), а `net/http` использует эту конфигурацию при приёме соединений.

## Минимальный HTTPS-сервер

Метод `ListenAndServeTLS` принимает два пути:

- `certFile` — файл сертификата сервера;
- `keyFile` — файл приватного ключа.

```go
package main

import (
    "errors"
    "log"
    "net/http"
    "time"
)

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("GET /", func(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("hello over https\n"))
    })

    server := &http.Server{
        Addr:              ":443",
        Handler:           mux,
        ReadHeaderTimeout: 2 * time.Second,
        ReadTimeout:       5 * time.Second,
        WriteTimeout:      10 * time.Second,
        IdleTimeout:       120 * time.Second,
    }

    err := server.ListenAndServeTLS("cert.pem", "key.pem")
    if err != nil && !errors.Is(err, http.ErrServerClosed) {
        log.Fatalf("https server failed: %s", err)
    }
}
```

`ListenAndServeTLS` загружает сертификат и ключ из файлов, создаёт TLS listener и начинает обслуживать HTTPS-запросы. Как и `ListenAndServe`, этот метод всегда возвращает ненулевую ошибку. При штатной остановке сервера через `Shutdown` вернётся [`http.ErrServerClosed`](https://pkg.go.dev/net/http#ErrServerClosed).

::: info
На Unix-based системах порт `:443` обычно требует повышенных прав. В локальных примерах часто используют непубличный порт, например `:8443`.
:::

## Сертификат и приватный ключ

Сертификат подтверждает имя сервера, а приватный ключ используется при TLS-рукопожатии. Файлы должны подходить друг к другу: сертификат должен быть выпущен для соответствующего приватного ключа.

```text
cert.pem  # сертификат сервера, обычно PEM-encoded
key.pem   # приватный ключ, обычно PEM-encoded
```

Для публичного домена сертификат обычно выпускает доверенный центр сертификации. Для локальных экспериментов можно использовать самоподписанный сертификат, но обычные браузеры и HTTP-клиенты не будут доверять ему без дополнительной настройки.

`ListenAndServeTLS` принимает именно пути к файлам. Если сертификаты нужно загружать иначе, например из секрет-хранилища или менять без перезапуска процесса, обычно настраивают [`http.Server.TLSConfig`](https://pkg.go.dev/net/http#Server.TLSConfig) и запускают сервер через TLS listener.

## Настройка TLSConfig

Поле `TLSConfig` позволяет передать серверу явную TLS-конфигурацию. Для базового HTTPS-сервера часто достаточно настроек по умолчанию, но минимальную версию протокола иногда фиксируют явно.

```go
package main

import (
    "crypto/tls"
    "errors"
    "log"
    "net/http"
    "time"
)

func main() {
    mux := http.NewServeMux()

    server := &http.Server{
        Addr:    ":443",
        Handler: mux,
        TLSConfig: &tls.Config{
            MinVersion: tls.VersionTLS12,
        },
        // Настройка таймаутов...
    }

    err := server.ListenAndServeTLS("cert.pem", "key.pem")
    if err != nil && !errors.Is(err, http.ErrServerClosed) {
        log.Fatalf("https server failed: %s", err)
    }
}
```

В этом примере сервер принимает TLS 1.2 и новее. TLS 1.3 будет использоваться автоматически, если его поддерживает клиент.

::: warning
Не копируйте случайные TLS-настройки из старых примеров. В Go стандартные значения уже учитывают современные безопасные настройки. Ручная настройка `CipherSuites`, `CurvePreferences` и других низкоуровневых полей нужна только при понятной политике совместимости.
:::

## Сертификаты в TLSConfig

Если не хочется передавать пути в `ListenAndServeTLS`, сертификат можно загрузить самостоятельно и передать в `TLSConfig.Certificates`.

```go
cert, err := tls.LoadX509KeyPair("cert.pem", "key.pem")
if err != nil {
    log.Fatalf("load certificate: %s", err)
}

server := &http.Server{
    Addr:    ":443",
    Handler: mux,
    TLSConfig: &tls.Config{
        MinVersion:   tls.VersionTLS12,
        Certificates: []tls.Certificate{cert},
    },
}

err = server.ListenAndServeTLS("", "")
if err != nil && !errors.Is(err, http.ErrServerClosed) {
    log.Fatalf("https server failed: %s", err)
}
```

Пустые строки в `ListenAndServeTLS("", "")` допустимы, если сертификаты уже заданы в `TLSConfig`. В этом варианте `net/http` не загружает файлы сертификата сам, а использует готовую конфигурацию TLS.

## HTTP/2

Когда сервер запущен с TLS, `net/http` автоматически включает поддержку HTTP/2 для клиентов, которые согласовали этот протокол через ALPN. В большинстве случаев для этого не требуется отдельный код.

Если нужно полностью контролировать набор протоколов, это делается через TLS-настройки сервера. Для обычного HTTPS-сервера лучше оставить поведение по умолчанию.
