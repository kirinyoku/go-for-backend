# Инициализация клиента

Пакет [`net/http`](https://pkg.go.dev/net/http) предоставляет два уровня работы с исходящими HTTP-запросами: пакетные функции вроде [`http.Get`](https://pkg.go.dev/net/http#Get) для быстрого старта и структуру [`http.Client`](https://pkg.go.dev/net/http#Client) для явной настройки поведения клиента.

`http.Client` не открывает соединения напрямую. Внутри него есть поле [`Transport`](https://pkg.go.dev/net/http#Transport), через которое клиент делегирует выполнение запроса сетевому уровню: открытие соединений, TLS, прокси и переиспользование TCP-соединений.

## Пакетные функции

Для коротких примеров, скриптов и ручных проверок удобно использовать пакетные функции [`http.Get`](https://pkg.go.dev/net/http#Get),[ `http.Post`](https://pkg.go.dev/net/http#Post) и [`http.Head`](https://pkg.go.dev/net/http#Head). Они позволяют выполнить запрос без явного создания клиента.

```go
package main

import "net/http"

func fetchStatus(url string) (int, error) {
    resp, err := http.Get(url)
    if err != nil {
        return 0, err
    }
    defer resp.Body.Close()

    return resp.StatusCode, nil
}
```

## Ограничения `http.DefaultClient`

Функции `http.Get`, `http.Post` и `http.Head` используют глобальный клиент [`http.DefaultClient`](https://pkg.go.dev/net/http#DefaultClient).

```go
var DefaultClient = &Client{}
```

`http.DefaultClient` — это клиент с настройками по умолчанию. Если его поле `Transport` равно `nil`, для сетевого уровня используется глобальный [`http.DefaultTransport`](https://pkg.go.dev/net/http#DefaultTransport).

Для небольших скриптов такой вариант удобен, но в приложении обычно используют явно настроенный клиент. Главная причина — контроль поведения. Например, у `http.DefaultClient` поле `Timeout` равно нулю, поэтому общий лимит времени на запрос не задан.

## Настройка `http.Client`

Структура `http.Client` позволяет задать настройки один раз и затем выполнять запросы через методы этого клиента.

```go
package main

import (
    "net/http"
    "time"
)

var client = &http.Client{
    Timeout: 10 * time.Second,
}
```

После инициализации простой `GET`-запрос выполняется методом [`client.Get`](https://pkg.go.dev/net/http#Client.Get):

```go
func fetchStatus(client *http.Client, url string) (int, error) {
    resp, err := client.Get(url)
    if err != nil {
        return 0, err
    }
    defer resp.Body.Close()

    return resp.StatusCode, nil
}
```

Такой код использует тот же удобный сценарий, что и `http.Get`, но поведение клиента теперь задается явно. Подробные различия между [`client.Timeout`](https://pkg.go.dev/net/http#Client.Timeout), context deadline и таймаутами `Transport` описаны в статье [Таймауты и контекст](./timeouts-and-context).

## Жизненный цикл клиента

`http.Client` рассчитан на переиспользование и безопасен для конкурентного использования несколькими goroutine. Обычно его создают один раз и передают туда, где выполняются исходящие запросы.

```go
type APIClient struct {
    httpClient *http.Client
}

func NewAPIClient() *APIClient {
    return &APIClient{
        httpClient: &http.Client{
            Timeout: 10 * time.Second,
        },
    }
}
```

::: warning
Создание нового `http.Client` с новым `http.Transport` для каждого запроса мешает переиспользованию соединений. Подробнее это разбирается в статье [Пул соединений](./connection-pooling).
:::
