# Настройка транспортного уровня

В то время как [`http.Client`](https://pkg.go.dev/net/http#Client) управляет логикой запросов высокого уровня, за сетевое взаимодействие отвечает «транспортный» уровень. Интерфейс [`http.RoundTripper`](https://pkg.go.dev/net/http#RoundTripper) является фундаментом этой системы: через него клиент настраивает TCP-соединения, TLS, прокси и дополнительное поведение вокруг отправки запроса.

## Интерфейс `RoundTripper`

`RoundTripper` — это интерфейс с одним методом [`RoundTrip`](https://pkg.go.dev/net/http#RoundTripper.RoundTrip). Он принимает запрос и возвращает ответ. Реализация по умолчанию — [`*http.Transport`](https://pkg.go.dev/net/http#Transport).

```go
type RoundTripper interface {
    RoundTrip(*Request) (*Response, error)
}
```

::: info 
Метод `RoundTrip` не должен интерпретировать ответ (например, проверять статус-код). Его задача — только доставить запрос и вернуть ответ или ошибку сетевого уровня. 
:::

## Настройка `http.Transport`

`http.Transport` — сложная структура, управляющая пулом соединений (Keep-Alive). Для изменения параметров рекомендуется **клонировать** [`http.DefaultTransport`](https://pkg.go.dev/net/http#DefaultTransport), чтобы сохранить безопасные стандартные настройки (таймауты подключения и TLS).

```go
transport := http.DefaultTransport.(*http.Transport).Clone()
transport.DisableCompression = true // Отключение автоматического Gzip

client := &http.Client{
    Transport: transport,
}
```

### Полезные настройки

|Поле|Назначение|
|:--|:--|
|[`ResponseHeaderTimeout`](https://pkg.go.dev/net/http#Transport.ResponseHeaderTimeout)|Ограничивает время ожидания заголовков ответа от сервера.|
|[`DisableCompression`](https://pkg.go.dev/net/http#Transport.DisableCompression)|Отключает автоматическую распаковку сжатых ответов (`gzip`).|
|[`Proxy`](https://pkg.go.dev/net/http#Transport.Proxy)|Позволяет направлять запросы через прокси-сервер.|
|[`TLSClientConfig`](https://pkg.go.dev/net/http#Transport.TLSClientConfig)|Настраивает параметры TLS и проверку сертификатов.|

::: tip 
Параметры управления пулом соединений (`MaxIdleConns`, `MaxIdleConnsPerHost` и др.) рассмотрены в [Пул соединений](./connection-pooling). 
:::

## Обертки над `RoundTripper`

Так как `http.Client` выполняет запрос через поле `Transport`, можно подставить туда собственную реализацию `RoundTripper`. Частый прием — обернуть один транспорт другим: внешний слой меняет или логирует запрос, а затем передает его дальше в настоящий сетевой транспорт.

Так удобно добавлять сквозное поведение для всех запросов конкретного клиента: авторизацию, логирование, метрики или повторные попытки. При этом сама отправка запроса остается за нижним транспортом, обычно за `http.DefaultTransport` или настроенным `*http.Transport`.

### Пример: Добавление API-ключа

```go
type AuthTransport struct {
    APIKey string
    Next   http.RoundTripper
}

func (t *AuthTransport) RoundTrip(req *http.Request) (*http.Response, error) {
    // Делаем копию запроса для модификации заголовков.
    req = req.Clone(req.Context())
    req.Header.Set("X-API-Key", t.APIKey)

    next := t.Next
    if next == nil {
        next = http.DefaultTransport
    }

    return next.RoundTrip(req)
}

func main() {
    client := &http.Client{
        Transport: &AuthTransport{
            APIKey: "secret",
            Next:   http.DefaultTransport,
        },
    }
    // Каждый запрос через этот клиент теперь содержит API-ключ.
}
```
